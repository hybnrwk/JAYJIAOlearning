# JAYJIAOlearning

这是一个以 Obsidian 为主要入口的个人学习知识库，也可以理解为一个长期在线的“我以后肯定用得到”现场。内容主要来自某财经院校 DS 专业学生的个人学习过程，用来归档课程笔记、教材课件、复习索引和跨课程概念卡片，覆盖计算机科学、数据科学、数学统计、经管课程与商业分析等方向。

它是一个长期维护的学习档案：原始资料按课程保存，Wiki 层负责提供稳定入口，Obsidian 负责双链、搜索、反向链接和日常复习。目录结构会随着学习进度和整理欲望波动，部分标题可能非常抽象；笔记质量也整体服从真实学习过程的物理规律：有些地方清楚，有些地方努力清楚，有些地方等未来的自己回来补。

## 仓库定位

这个仓库主要服务三件事：

- 资料归档：保存课程笔记、课件、教材 PDF、作业相关资料和图片附件。
- 快速检索：通过课程索引、Wiki 首页和 Obsidian 搜索，快速回到需要的知识点。
- 复习串联：用概念卡片和双链关系，把不同课程里反复出现的内容连起来。

适合用来：

- 查找某门课的笔记、教材、课件或复习材料。
- 从 `00-Wiki/` 进入学科地图和高频概念卡片。
- 用 Obsidian 查看笔记之间的链接、反链和局部图谱。
- 作为个人学习过程的备份与同步仓库。

不适合用来：

- 直接作为标准答案、官方讲义或可引用资料源。
- 期待统一的构建、运行、部署或 API 接口。
- 直接相信所有笔记内容而不核对原书、课件或更可靠来源。

## 用 Obsidian 使用这个知识库

### 打开方式

1. 克隆或下载本仓库到本地。
2. 打开 Obsidian，选择 `Open folder as vault`。
3. 选择仓库根目录，也就是包含 `README.md`、`00-Wiki/`、`cs_ds/`、`数统/` 的这一层目录。
4. 打开 [`00-Wiki/Home.md`](00-Wiki/Home.md)，可以把它加入书签，作为知识库首页使用。

不要只把 `00-Wiki/` 或某个课程子目录单独作为 vault 打开。这样会丢失大量跨目录链接、图片引用和课程索引关系。

### 首次设置建议

这个仓库没有依赖特定的 Obsidian 主题或插件，直接打开即可使用。为了减少新笔记和附件路径混乱，建议检查这些设置：

- 使用 Obsidian 默认的 `[[Wiki links]]` 链接格式。
- 新附件默认保存到 `images/`，便于统一管理截图和粘贴图片。
- 新建内部链接时优先使用相对稳定的短路径或最短路径，避免移动笔记后链接过长。
- 如果启用同步插件或第三方同步工具，先确认不会自动提交临时文件、缓存文件和超大附件。

### 日常使用方式

- 找课程资料：从 [`00-Wiki/Home.md`](00-Wiki/Home.md) 或对应目录的 `00-*索引.md` 进入。
- 找复习重点：从 [`00-Wiki/Review.md`](00-Wiki/Review.md) 进入，再回到课程索引和原始笔记。
- 找概念关系：打开 [`00-Wiki/Concepts/`](00-Wiki/Concepts/)，配合反向链接和局部图谱查看相关课程内容。
- 找具体文件：用 Obsidian 全局搜索课程名、知识点、教材名或文件名；PDF 和图片也可以在文件列表中直接定位。
- 写新笔记：优先放回对应课程目录，再在课程索引或概念卡片中补一个链接。
- 整理旧笔记：优先补链接、补索引、补概念卡片，不必为了整齐而大规模移动原始资料。

### 推荐的维护习惯

- 课程正文放在课程目录，Wiki 层只做导航、地图和概念入口。
- 高频、跨课程、容易忘的知识点可以沉淀成 `00-Wiki/Concepts/` 下的短卡片。
- 每门课尽量维护一个 `00-*索引.md`，让后续复习不用从文件树里硬翻。
- 新增 PDF、图片或 Office 文件前，先确认文件大小、来源和是否真的需要纳入仓库。
- 如果只是临时数据、代码中间产物、压缩包或视频，通常不要提交。

## 大文件与 Git LFS

部分大型 PDF 使用 Git LFS 管理。克隆仓库后如需完整获取这些文件，请确保本机已安装 Git LFS，并执行：

```bash
git lfs install
git lfs pull
```

新增文件时建议注意：

- 超过 100 MB 的文件不要直接用普通 Git 提交。
- 大型 PDF 应优先使用 Git LFS 管理。
- 视频、压缩包、临时文件和大体积中间产物通常不应提交到仓库。
- `.gitignore` 已忽略一批常见无关文件，但新增大文件前仍建议手动检查文件大小和用途。

## 内容边界

- 笔记来自个人理解、课堂记录、阶段性整理、考前急救和一些精神状态不明的夜晚，可能有错、有漏、有重复，也可能已经过时。
- 有些笔记确实“又臭又长”：它们可能逻辑绕远、结构松散、标题很认真、正文很意识流。能看懂是缘分，看不懂也不一定是你的问题。
- 仓库里的课件、教材摘录、讲义、图片等主要用于个人学习和备份；相关版权归原作者、课程团队、出版社或其他权利方所有。
- 这里的内容不构成学术、职业、投资、法律或任何正式建议。
- 如果发现不适合公开、引用不当或其他问题，应后续修正、删除或重新整理。

这个仓库的核心价值不是形式完美，而是能在需要的时候把资料找回来、把知识点串起来、把复习路径接上。
