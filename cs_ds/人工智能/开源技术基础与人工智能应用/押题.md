## 4. CNN 相比普通 FNN 更适合图像任务，主要因为 CNN 具有

## 5. RNN 的“记忆能力”主要来自

## 6. 原始 RNN 在长序列任务中的主要问题是

## 2. 论述神经网络训练为什么需要梯度下降和反向传播

**参考答案要点：**

```text
1. 神经网络训练的目标是最小化损失函数。
2. 损失函数衡量预测结果与真实标签之间的差距。
3. 梯度下降根据损失函数梯度的反方向更新参数。
4. 学习率决定每次更新步长，太大易震荡，太小收敛慢。
5. 神经网络参数很多，直接计算梯度代价高。
6. 反向传播利用链式法则，从输出层向输入层逐层计算梯度。
7. 反向传播负责算梯度，梯度下降负责用梯度更新参数。
8. 二者共同构成深度学习模型训练的基础。
```

**解析：**  
这题是“AI 理论基础”常见论述。关键词：**损失函数、梯度、学习率、链式法则、参数更新**。

```text
7. Transformer 可扩展性强，能支撑 GPT、BERT、DeepSeek 等模型。
8. 因此它成为大模型时代的核心架构。
```

**解析：**

## 8. 论述数据清洗、数据标注与模型训练效果之间的关系

**参考答案要点：**

```text
1. 原始数据可能存在缺失、重复、噪声、异常和格式不一致。
2. 数据清洗提高数据标准化、准确性和一致性。
3. 数据标注给原始数据添加类别、位置、语义、情感等标签。
4. 监督学习依赖带标签样本学习规律。
5. 错误标注会让模型学习错误规律。
6. 图像框偏移会影响目标检测，语音时间轴错位会影响语音识别，文本实体边界错误会影响 NLP。
7. 高质量数据清洗和标注能提升模型准确性、稳定性和泛化能力。
8. 因此数据处理质量是 AI 系统效果的重要基础。
```

**解析：**  
这题和学姐提示的“前后端数据清洗”很贴合。答的时候要强调：**清洗在前，标注在后，训练效果受二者共同影响**。

# 三、大题 4 道

## 大题 1：系统分析 Transformer 相比 RNN 的优势，并画出答题框架

### 参考答案

RNN 和 Transformer 都可以用于序列建模，但二者的架构差异决定了它们在长文本、大规模训练中的表现差异。

RNN 的核心是循环结构。它按照时间步依次处理输入序列，当前时刻的隐藏状态依赖当前输入和上一时刻隐藏状态。因此，RNN 具有一定记忆能力，适合文本、语音和时间序列等任务。但是 RNN 的缺点也很明显。第一，它必须按顺序处理序列，不能很好地并行训练；第二，长序列中信息需要经过很多时间步传递，容易出现梯度消失或梯度爆炸；第三，它对长距离依赖的建模能力较弱。

Transformer 的核心是自注意力机制。自注意力机制让序列中的每个位置都可以直接关注其他所有位置，而不是依赖一步步传递。具体来说，每个 token 会被映射为 Query、Key、Value。Query 与 Key 计算注意力分数，注意力分数再作为权重对 Value 加权求和，从而得到融合上下文的新表示。这样，句子中距离很远但语义相关的词也可以直接建立联系。

Transformer 还引入多头注意力机制。多头注意力让模型在多个表示子空间中同时学习不同类型的信息，例如语法关系、指代关系、局部搭配关系、主题关系等。多个头的输出拼接后再经过线性变换，使模型获得更丰富的语义表达。

因此，Transformer 相比 RNN 的优势主要有：第一，支持并行训练，效率更高；第二，能够更好捕捉长距离依赖；第三，多头注意力提高了表达能力；第四，架构更适合扩展到大规模参数和海量数据。因此，Transformer 成为 GPT、BERT、DeepSeek、CLIP、ViT 等现代大模型的基础架构。

### 答题框架

```text
1. 先讲 RNN：循环结构、按时间步处理、有记忆能力。
2. 再讲 RNN 缺点：不能并行、梯度消失、长距离依赖弱。
3. 再讲 Transformer：自注意力、QKV、多头注意力。
4. 最后总结优势：并行训练、长距离依赖、表达能力强、适合大模型。
```

### 解析

这是最高优先级大题。考试如果问“Transformer 相比 RNN 最大优势是什么”，重点压缩成：

```text
最大优势是自注意力机制带来的并行计算能力和长距离依赖建模能力。
```

## 大题 2：从人工智能发展史角度，说明 AI 如何从规则推理发展到大模型和 Agent

### 参考答案

人工智能的发展可以看作从规则驱动到数据驱动，再到大模型驱动和智能体驱动的过程。

早期人工智能主要是符号主义路线。符号主义认为智能可以通过符号表示、逻辑规则和推理过程实现。例如逻辑理论家、通用问题求解器和专家系统，都试图把人类知识转化为明确规则。专家系统采用“知识库 + 推理机”的结构，在医疗、化学等垂直领域曾取得一定成功。

但是专家系统存在明显局限。它需要人工提取专家知识，知识获取成本高；规则数量增加后维护困难；面对新场景时泛化能力较弱。因此，人工智能逐渐从人工写规则转向机器从数据中学习。

机器学习和深度学习阶段，模型通过数据学习规律。神经网络通过输入层、隐藏层、输出层构建可训练函数，利用反向传播计算梯度，用梯度下降更新参数。CNN 推动了图像识别，RNN 推动了序列建模，深度学习借助大规模数据和 GPU 算力显著提高了 AI 的感知能力。

2017 年 Transformer 出现后，人工智能进入大模型时代。Transformer 通过自注意力机制解决了 RNN 难以并行和长距离依赖弱的问题。基于 Transformer 的 GPT、BERT 等模型采用“预训练 + 微调”范式，在海量文本上学习通用知识，再适配具体任务。ChatGPT 的出现让生成式 AI 进入大众视野，DeepSeek-R1 等推理模型进一步强化了数学、代码和复杂逻辑能力。

在大模型基础上，AI Agent 成为新阶段。普通大模型主要生成内容，而 Agent 能围绕目标进行感知、规划、工具调用和行动。它结合 LLM、RAG、Function Calling、ReAct、MCP、向量数据库等技术，使 AI 从“回答问题”走向“完成任务”。

因此，人工智能的发展脉络可以概括为：符号主义和专家系统强调人工规则，机器学习和深度学习强调数据学习，Transformer 和大模型强调通用预训练能力，而 AI Agent 则进一步强调自主行动和任务完成能力。

### 解析

这题适合考“人工智能发展史和理论”。答题要有阶段感：

```text
符号主义 → 专家系统 → 机器学习 → 深度学习 → Transformer → 大模型 → Agent
```

## 大题 3：设计一个“课程复习 AI Agent”，说明其模块和工作流程

### 参考答案

可以设计一个“课程复习 AI Agent”，目标是帮助学生基于课程 PPT 进行知识梳理、重点提取、答疑和出题训练。

第一，感知模块。该模块负责接收和理解用户输入，包括 PPT、PDF、Word、图片截图、学生提问等。如果输入是文本课件，系统可以直接解析文字；如果是图片截图，可以使用 OCR 提取文字；如果用户提出问题，系统需要识别用户意图，例如“总结这一章”“解释 Transformer”“帮我出题”。

第二，认知与决策模块。该模块以大语言模型为核心，负责理解课程内容、分析用户需求、规划答题结构。例如用户要求“复习第五讲”，Agent 需要判断应先提取章节目录，再整理核心概念，最后生成可能考题。对于复杂概念，可以使用 CoT 进行分步解释；对于需要查找课件内容的问题，可以调用知识库检索。

第三，记忆模块。短期记忆保存当前对话上下文和正在复习的章节；长期记忆保存课程知识库、学生薄弱点、错题记录和复习进度。长期知识可以通过向量数据库实现，当用户提问时，系统先检索相关课件片段，再交给模型生成答案，这就是 RAG。

第四，行动模块。行动模块负责执行具体操作，例如生成复习提纲、生成选择题和论述题、整理错题、导出 Markdown 或 Word 文档、调用搜索工具核查概念、调用代码工具生成表格等。

第五，工具调用机制。对于需要外部工具的任务，可以使用 Function Calling。模型先判断需要调用什么工具，例如“检索课件”“生成题目”“导出文档”，然后输出函数名和参数，由外部程序执行，再把结果返回模型。对于复杂多工具场景，可以使用 MCP 统一管理模型与外部工具、数据源之间的交互。

第六，安全与质量控制。Agent 不应编造课件没有的内容；对不确定信息应提示用户；生成题目时要标明答案和解析；涉及文件删除、发送邮件等高风险操作时应要求用户确认。

该 Agent 的工作流程可以概括为：用户输入目标 → 感知模块理解材料和需求 → RAG 检索课件内容 → LLM 规划复习结构 → 调用工具生成题目或文档 → 用户反馈 → Agent 继续优化。

### 解析

这题本质考 AI Agent。高分关键词：

```text
感知模块
认知决策模块
行动模块
记忆模块
RAG
Function Calling
ReAct
MCP
短期记忆
长期记忆
向量数据库
```

## 大题 4：结合数据清洗和数据标注，设计一个用于训练 YOLO 的图像数据集建设流程

### 参考答案

如果要训练一个 YOLO 目标检测模型，例如识别道路中的车辆和行人，需要先建设高质量图像数据集。完整流程包括数据采集、数据清洗、数据标注、质量检测、格式转换和模型训练。

第一步是数据采集。可以从车载摄像头、道路监控、公开自动驾驶数据集等来源收集图片或视频帧。采集时应保证场景多样性，包括白天、夜晚、晴天、雨天、城市道路、高速道路、拥堵场景等。这样模型才能学习到不同环境下车辆和行人的特征。

第二步是数据清洗。原始图片可能存在模糊、重复、损坏、分辨率过低、无关场景等问题，需要删除或修正。还要统一图片格式和命名规则。对于涉及隐私的信息，如清晰人脸、车牌等，可以根据任务需要进行脱敏处理。数据清洗可以提高后续标注质量。

第三步是制定标注规范。需要明确标注对象，例如车辆、行人、交通灯、交通标志等；明确遮挡、截断、重叠目标如何处理；明确矩形框是否贴合目标可见部分；明确不同类别标签名称。

第四步是使用标注工具进行图像标注。可以使用 LabelMe、LabelImg、CVAT 等工具。对于 YOLO 目标检测任务，最常见的是矩形框标注。标注人员需要对每个目标画框，并赋予正确类别。框应尽量贴合目标边缘，不能过大或过小，避免漏标、错标和重复标。

第五步是质量检测。需要抽检标注结果，检查框是否贴合目标、类别是否正确、是否漏标、遮挡目标是否按规范处理。如果错误较多，应返工修改。高质量标注对模型效果非常重要，错误标签会导致模型学习错误规律。

第六步是格式转换和数据集划分。LabelMe 默认输出 JSON 文件，LabelImg 可输出 Pascal VOC 或 YOLO 格式。训练 YOLO 时，需要将标注数据转换成 YOLO 支持的格式，并划分训练集、验证集和测试集。

第七步是模型训练与评估。使用 YOLO 加载数据集进行训练，通过准确率、召回率、mAP 等指标评估效果。如果模型表现不好，需要检查数据是否不足、标注是否错误、类别是否不均衡，并继续补充和优化数据。

### 解析

这题把“数据清洗 + 数据标注 + YOLO”串起来，是第六讲最可能的大题形式之一。

答题框架：

```text
采集 → 清洗 → 规范 → 标注 → 质检 → 格式转换 → 训练评估
```

高分关键词：

```text
LabelMe
LabelImg
YOLO
矩形框标注
JSON
COCO
训练集/验证集/测试集
漏标
错标
重复标
标注质量影响模型效果
```

# 二、论述题 6 道

## 1. 论述人工智能的发展历程，并说明大模型时代的特点

**参考答案：**

人工智能的发展大致可以分为几个阶段。第一阶段是符号主义阶段，强调用规则、符号和逻辑推理实现智能，代表是逻辑理论家、通用问题求解器和专家系统。第二阶段是专家系统阶段，采用“知识库 + 推理机”的方式解决特定领域问题，但由于知识获取困难、规则维护成本高、泛化能力弱，逐渐进入低谷。第三阶段是机器学习和深度学习阶段，人工智能从人工写规则转向从数据中学习规律，神经网络、反向传播、CNN、RNN 等方法推动了图像识别、语音识别和自然语言处理的发展。第四阶段是大模型阶段，Transformer 架构解决了长距离依赖和并行训练问题，GPT、BERT、ChatGPT、DeepSeek 等模型使 AI 具备了更强的语言理解、生成、推理和工具调用能力。

大模型时代的特点是：模型参数规模大，训练数据规模大，计算资源需求高；采用“预训练 + 微调”的范式；具备文本生成、代码生成、推理、多模态理解等能力；开源社区和商业闭源模型共同推动生态发展；AI 的应用从“识别和分析”扩展到“生成内容”和“执行任务”。

**解析：**

## 2. 论述 Transformer 相比 RNN 的优势

**参考答案：**

RNN 适合处理序列数据，它通过隐藏状态把前一时间步的信息传递到当前时间步，因此具有一定记忆能力。但是 RNN 需要按顺序逐步处理序列，难以并行训练；当序列较长时，容易出现梯度消失或梯度爆炸，导致模型难以捕捉长距离依赖。

Transformer 的核心是自注意力机制。自注意力允许序列中的每个位置直接与其他所有位置交互，通过 Query、Key、Value 计算注意力权重，并根据权重聚合上下文信息。这使模型能够直接建立远距离词语之间的联系。多头注意力进一步让模型在多个表示子空间中同时捕捉不同类型的信息。

因此，Transformer 相比 RNN 的优势主要包括：第一，支持并行计算，训练效率更高；第二，更容易捕捉长距离依赖；第三，更适合大规模数据和大参数模型；第四，它成为 GPT、BERT、CLIP、ViT 等现代大模型和多模态模型的基础架构。

**解析：**  
这是最像学长学姐提到的“最后一道大题”。如果只问“最大优势是什么”，可以答：**自注意力机制带来的并行训练能力和长距离依赖建模能力**。

## 3. 论述开源社区如何推动大模型发展

**参考答案：**

开源社区推动大模型发展主要体现在三个方面。第一，开源降低了技术门槛。PyTorch、TensorFlow 等开源框架封装了自动求导、反向传播、GPU 加速等底层能力，使普通开发者不用从零实现深度学习算法。第二，开源模型和开源数据降低了研发成本。LLaMA、Mistral、Qwen、DeepSeek 等模型开放权重或代码后，研究者和开发者可以基于已有基础模型进行微调、部署和应用开发，而不必从零预训练。第三，开源社区促进快速迭代。全球开发者可以共同测试模型、发现问题、修复 Bug、贡献工具和教程，形成模型、数据、工具、应用平台之间的生态闭环。

在大模型时代，开源的意义已经不只是代码开源，还包括模型权重、训练脚本、推理框架、数据集、评测工具、Agent 框架和应用平台的开放。Hugging Face 和 ModelScope 这类平台也推动了模型、数据和应用的共享。

## 4. 论述 AI Agent 与普通生成式 AI 的区别

**参考答案：**

普通生成式 AI 主要根据用户提示生成文本、图像、代码、音频、视频等内容，常见形式是一问一答或者一次性生成。它的重点是“生成内容”。AI Agent 则更强调“完成任务”。它以大语言模型为认知核心，具备感知环境、理解目标、自主规划、调用工具、执行动作、根据反馈调整策略等能力。

AI Agent 通常包括感知模块、认知决策模块、行动模块和记忆模块。感知模块负责接收用户指令、文档、图像、语音等外部信息；认知决策模块负责理解任务、推理和规划；行动模块负责调用搜索、数据库、API、代码执行器等工具；记忆模块负责保存当前上下文、历史经验和长期知识。相比普通生成式 AI，Agent 更适合复杂、多步骤、需要外部工具和动态反馈的任务。

例如，普通生成式 AI 可以告诉用户“如何查询订单”，而客服 Agent 可以识别用户意图、调用订单数据库、查询物流状态，并返回处理结果。

**解析：**  
这题是第五讲重点。答题不要只说“Agent 更智能”，要突出：**目标导向、感知、规划、行动、工具调用、记忆、反馈闭环**。

## 5. 论述 AIGC 的概念、应用与风险

**参考答案：**

AIGC 即 AI Generated Content，指利用人工智能生成文本、图像、音频、视频等内容。它属于生成式 AI 的具体应用形式。文本 AIGC 可以用于新闻写作、报告总结、文案生成、代码生成；图像 AIGC 可以用于海报设计、AI 绘画、电商商品图、AI 试衣；音频 AIGC 可以用于语音合成、语音克隆、有声书和配音；视频 AIGC 可以用于文生视频、图生视频、自动剪辑、数字人和影视特效。

AIGC 的价值主要是降本增效和创新体验。它可以降低内容生产成本，提高创意迭代速度，并让非专业用户也能完成过去需要专业技能的内容创作。但是 AIGC 也存在风险，包括版权争议、虚假信息、深度伪造、语音冒用、隐私泄露、内容质量不可控等。因此，在使用 AIGC 时需要注意授权、标注 AI 生成内容、加强审核和建立合规机制。

## 6. 论述数据清洗和数据标注在人工智能中的作用

**参考答案：**

数据清洗和数据标注是人工智能模型训练前的重要数据处理环节。数据清洗的作用是识别并处理原始数据中的缺失值、噪声、重复记录、错误格式和异常值，使数据更加标准、准确和一致。常见方法包括缺失值填充或删除、噪声平滑、重复数据去重、格式统一等。

数据标注则是给原始数据添加机器可学习的标签。例如，在图像中给车辆和行人画框，在语音中标出对应文字和时间轴，在文本中标出实体、关系、情感或意图。对于监督学习而言，模型需要从带标签样本中学习输入与输出之间的规律，因此标注数据是训练模型的重要基础。

二者关系密切。清洗质量影响标注质量，标注质量影响模型训练效果。如果数据未清洗，标注人员可能基于错误或重复样本进行标注；如果标注错误，模型会学习错误规律，导致识别偏差。因此，一个完整的数据生产流程应包括数据采集、数据清洗、制定标注规范、人工标注、质量检测、返工修正和格式导出。

**解析：**  
这题对应第六讲，也契合学姐说的“前后端、数据清洗”要注意。答题关键词：**缺失值、噪声、重复、格式标准化、标注标签、监督学习、质量影响模型效果**。

# 三、大题 2 道

## 大题 1：Transformer 相比 RNN 的最大优势是什么？请结合架构顺便分析 Transformer 架构

### 参考答案

RNN 和 Transformer 都可以处理序列数据，但二者的建模方式不同。RNN 的核心是循环结构，它按时间步依次处理输入序列。当前时刻的隐藏状态依赖当前输入和上一时刻的隐藏状态，因此 RNN 具有一定记忆能力，适合处理文本、语音、时间序列等任务。但是 RNN 存在两个明显问题：第一，它必须按顺序处理序列，难以并行训练；第二，当序列较长时，早期信息需要经过很多时间步传递，容易出现梯度消失或梯度爆炸，导致长距离依赖建模能力弱。

Transformer 的最大优势在于自注意力机制。自注意力机制允许序列中的每个位置在处理自身表示时，直接关注序列中所有其他位置的信息。具体来说，每个输入向量会被映射成 Query、Key、Value 三类向量。模型用 Query 和其他位置的 Key 计算注意力分数，再根据这些分数对 Value 进行加权求和，得到融合上下文的新表示。这样，一个词可以直接关注句子中距离很远但语义相关的词，不需要像 RNN 那样一步步传递信息。

Transformer 还使用多头注意力机制。多头注意力会在多个不同表示子空间中并行计算注意力，让不同注意力头关注不同信息，例如语法关系、指代关系、主题关系、局部搭配关系等。多个头的输出拼接后再经过线性变换，形成更丰富的上下文表示。

从整体架构看，Transformer 通常由编码器和解码器组成。编码器负责理解输入序列，解码器负责生成输出序列。每一层中通常包括多头自注意力、前馈神经网络、残差连接和层归一化等结构。自注意力负责上下文交互，前馈网络负责非线性变换，残差连接和层归一化有助于稳定深层网络训练。

因此，Transformer 相比 RNN 的最大优势可以概括为：第一，自注意力机制使模型能够直接捕捉长距离依赖；第二，序列中不同位置可以并行计算，训练效率更高；第三，多头注意力增强了模型表达能力；第四，Transformer 更适合大规模数据、大参数模型和 GPU 并行训练。正因为这些优势，Transformer 成为 GPT、BERT、CLIP、ViT、DeepSeek 等现代大模型的重要基础架构。

### 解析

这道题是最应该背的大题。它的标准结构是：

```text
先讲 RNN
再讲 RNN 局限
再讲 Transformer 自注意力
再讲 QKV
再讲多头注意力
最后总结优势
```

高分关键词：

```text

Q、K、V
多头注意力
并行训练

```

## 大题 2：人工智能系统从“数据—算法—模型—应用”角度如何构成？请结合神经网络、数据清洗、数据标注、大模型和 Agent 说明

### 参考答案

一个完整的人工智能系统可以从“数据—算法—模型—应用”四个层次理解。

第一层是数据。数据是 AI 的基础资源。原始数据可能来自日志、传感器、网页、业务系统、图片、语音、文本等来源，但原始数据往往存在缺失、重复、噪声、异常和格式不统一等问题，因此需要进行数据清洗。数据清洗包括缺失值处理、噪声处理、重复数据去重、格式标准化等。清洗后的数据还需要根据任务进行数据标注，例如图像中的矩形框、关键点、多边形标注，语音中的转写和时间轴标注，文本中的实体、关系、情感和意图标注。标注数据为监督学习提供了“标准答案”，其质量直接影响模型训练效果。

第二层是算法。人工智能算法包括传统机器学习方法和深度学习方法。神经网络是深度学习的重要基础。FNN 是前馈神经网络，信息单向传播，适合一般分类和回归任务；CNN 通过卷积层和池化层提取图像局部特征，适合图像识别和目标检测；RNN 通过循环结构处理序列数据，适合文本、语音和时间序列，但存在长距离依赖和梯度问题。为了训练神经网络，需要使用反向传播高效计算梯度，再用梯度下降更新参数。

第三层是模型。随着数据、算力和算法的发展，人工智能从普通神经网络发展到大模型。Transformer 通过自注意力机制解决了 RNN 难以并行和长距离依赖弱的问题，成为大语言模型的核心架构。大语言模型通常采用“预训练 + 微调”范式，先在海量数据上学习通用能力，再通过微调适配具体任务。DeepSeek-R1 这类推理模型进一步强化了数学、代码和逻辑推理能力。LoRA、QLoRA 等参数高效微调技术降低了大模型定制成本。

第四层是应用。AIGC 代表 AI 从“分析内容”走向“生成内容”，可以生成文本、图像、音频、视频，应用于电商、广告、新闻、影视、游戏等领域。AI Agent 则进一步代表 AI 从“生成内容”走向“完成任务”。Agent 以大语言模型为认知核心，结合感知模块、决策模块、行动模块和记忆模块，能够理解目标、规划步骤、调用工具、执行动作并根据反馈调整。Function Calling、ReAct、RAG、向量数据库和 MCP 等技术使 Agent 能够连接外部工具和知识库。

综上，人工智能系统不是单一模型，而是由数据基础、算法机制、模型能力和应用场景共同构成。数据清洗和标注决定训练材料质量，神经网络和 Transformer 决定模型学习能力，大模型增强理解、生成和推理能力，AIGC 和 Agent 则将 AI 能力转化为现实应用。

### 解析

这道题是综合大题，适合考“人工智能 AI 各部分的不同理论和技术，前后端数据清洗”。答题时要把课件串起来，而不是只讲某一章。

