# 1. 第一重点：人工智能发展史

这是最适合出论述题的部分。你要会讲“AI 是怎么一路发展到今天的大模型和 Agent 的”。

## 1.1 必背主线

```text
人工智能诞生
    ↓
符号主义
    ↓
专家系统
    ↓
第一次 / 第二次 AI 低谷
    ↓
机器学习
    ↓
深度学习
    ↓
Transformer
    ↓
大语言模型
    ↓
AIGC
    ↓
AI Agent
```

## 1.2 每个阶段抓什么

|阶段|核心思想|代表内容|局限 / 意义|
|---|---|---|---|
|人工智能诞生|机器能否表现智能|图灵测试、达特茅斯会议|AI 作为学科出现|
|符号主义|用符号、规则、逻辑推理模拟智能|LISP、逻辑推理程序|依赖人工规则，泛化弱|
|专家系统|把专家知识整理为规则|知识库 + 推理机|知识获取困难，维护成本高|
|机器学习|从数据中学习规律|分类、回归、SVM、随机森林|不再完全依赖人工规则|
|深度学习|用多层神经网络自动提取特征|CNN、RNN、BP、AlexNet|需要数据和算力|
|Transformer|自注意力机制|GPT、BERT 的基础|解决并行和长依赖问题|
|大模型|预训练 + 微调，通用能力增强|GPT、BERT、DeepSeek|AI 进入生成、推理阶段|
|AIGC|AI 生成内容|文本、图像、音频、视频|降本增效，也有风险|
|Agent|AI 执行任务|RAG、工具调用、MCP|从“回答”走向“行动”|

## 1.3 关键概念

### 图灵测试

```text
如果测试者通过文字对话无法判断对方是人还是机器，就可以认为机器表现出智能。
```

### 符号主义

```text
符号主义认为智能可以通过符号表示、逻辑规则和推理过程实现。
```

### 专家系统

```text
专家系统 = 知识库 + 推理机
```

知识库保存专家规则，推理机根据规则进行判断。

### AI 低谷的原因

```text
预期过高
算力不足
数据不足
任务复杂度被低估
专家系统维护成本高
泛化能力差
商业价值不如预期
```

### 深度学习爆发原因

```text
大规模数据
GPU 算力提升
神经网络结构改进
反向传播成熟
开源框架普及
```

## 1.4 论述题模板：人工智能发展史

这段建议直接背：

```text
人工智能的发展可以看作从规则驱动到数据驱动，再到大模型驱动和智能体驱动的过程。早期人工智能主要采用符号主义路线，认为智能可以通过符号、规则和逻辑推理实现。专家系统进一步把专家知识整理成“知识库 + 推理机”的形式，用于医学、化学等特定领域，但它存在知识获取困难、维护成本高、缺乏常识和泛化能力弱等问题。

后来人工智能逐渐从人工写规则转向从数据中学习，机器学习和深度学习成为主流。神经网络通过反向传播和梯度下降从大量数据中自动学习特征，CNN 推动了图像识别，RNN 推动了序列建模，ImageNet 和 AlexNet 则引爆了深度学习热潮。

2017 年 Transformer 架构提出后，自注意力机制解决了 RNN 难以并行和长距离依赖弱的问题，GPT、BERT、ChatGPT、DeepSeek 等大模型快速发展。大模型通过“预训练 + 微调”获得通用语言理解、生成和推理能力。进一步地，AIGC 让 AI 可以生成文本、图像、音频、视频，而 AI Agent 则让 AI 从被动回答问题走向主动规划、调用工具和完成任务。
```

# 2. 第二重点：Transformer vs RNN

这是最需要背的大题。学长说往年大题最后一道考了 Transformer 架构，特别是“Transformer 相比 RNN 的最大优势是什么”。

## 2.1 RNN 是什么

RNN，即循环神经网络，用来处理序列数据。

核心特点：

```text
当前时刻的输出不仅依赖当前输入，还依赖前一时刻隐藏状态。
```

也就是：

```text
x1 → h1 → x2 → h2 → x3 → h3 ...
```

适合：

```text
文本
语音
时间序列
股票价格
传感器数据
```

## 2.2 RNN 的问题

必须会写这三点：

```text
1. 串行计算，难以并行训练。
2. 长序列中容易梯度消失或梯度爆炸。
3. 长距离依赖建模能力弱。
```

解释一下：

RNN 处理句子时，要一个词一个词往后传。前面很远的信息要经过很多步才能影响后面的词，所以容易丢失。

例如：

```text
我昨天在图书馆借了一本关于人工智能的书，虽然它很厚，但内容很有趣。
```

模型要知道“它”指的是“书”，需要保留很远之前的信息。RNN 做这件事比较困难。

## 2.3 Transformer 的核心：自注意力机制

Transformer 的核心不是循环，而是：

```text
Self-Attention 自注意力机制
```

它的思想：

```text
序列中的每个 token 都可以直接关注其他所有 token。
```

这就避免了 RNN 那种一步步传递信息的问题。

## 2.4 Q、K、V 怎么理解

|名称|全称|直观含义|
|---|---|---|
|Q|Query|我现在想找什么信息|
|K|Key|每个词提供的索引标签|
|V|Value|每个词真正携带的信息内容|

计算过程：

```text
1. 当前 token 产生 Query；
2. 其他 token 产生 Key 和 Value；
3. Query 和 Key 计算匹配程度；
4. 匹配程度变成注意力权重；
5. 用权重对 Value 加权求和；
6. 得到融合上下文的新表示。
```

## 2.5 多头注意力是什么

一个注意力头只能从一个角度看句子，多头注意力可以让模型从多个角度同时理解。

不同头可能关注：

```text
语法关系
主谓宾关系
指代关系
局部搭配
因果关系
主题关系
情感关系
```

一句话：

```text
多头注意力让模型在多个表示子空间中并行学习不同类型的信息关系。
```

## 2.6 Transformer 相比 RNN 的优势

必须背：

|优势|解释|
|---|---|
|并行训练|不需要按时间步逐个处理，训练效率更高|
|长距离依赖更强|任意两个 token 可以直接交互|
|表达能力更强|多头注意力能捕捉不同关系|
|可扩展性更好|适合大数据、大参数、大模型|
|成为大模型基础|GPT、BERT、DeepSeek、CLIP、ViT 都依赖 Transformer 思路|

## 2.7 大题模板：Transformer 相比 RNN 的优势

这段建议重点背：

```text
RNN 通过循环结构处理序列数据，当前隐藏状态依赖当前输入和前一时刻隐藏状态，因此具有一定记忆能力，适合文本、语音和时间序列任务。但是 RNN 必须按时间步顺序处理序列，难以并行训练；当序列较长时，信息需要经过很多步传递，容易出现梯度消失或梯度爆炸，导致长距离依赖建模能力较弱。

Transformer 的核心是自注意力机制。自注意力机制允许序列中的每个位置直接关注其他所有位置，通过 Query、Key、Value 计算注意力权重，并对 Value 进行加权求和，从而得到融合上下文的信息表示。这样，距离很远但语义相关的词也可以直接建立联系，不需要像 RNN 那样一步步传递隐藏状态。

此外，Transformer 使用多头注意力机制，让模型在多个表示子空间中同时捕捉不同类型的信息，例如语法关系、指代关系、局部搭配关系和主题关系。由于 Transformer 不依赖循环结构，因此更容易并行训练，也更适合 GPU 加速和大规模数据训练。

因此，Transformer 相比 RNN 的最大优势在于：一是支持并行训练，训练效率更高；二是更容易捕捉长距离依赖；三是多头注意力增强了模型表达能力；四是可扩展性强，适合作为 GPT、BERT、DeepSeek 等现代大语言模型的基础架构。
```

## 2.8 如果考试只问一句“最大优势是什么”

答：

```text
Transformer 相比 RNN 的最大优势是自注意力机制带来的并行训练能力和长距离依赖建模能力。RNN 需要按时间步串行处理序列，而 Transformer 可以让每个 token 直接关注其他所有 token，因此更适合长文本和大规模模型训练。
```

# 3. 第三重点：神经网络基础

这一部分会出选择题，也可能融入综合大题。

## 3.1 神经网络是什么

神经网络可以理解为：

```text
一个由输入层、隐藏层、输出层构成的可训练函数。
```

输入可以是：

```text
图片
文本
音频
表格数据
传感器数据
```

输出可以是：

```text
类别
数值
文本
图片
语音
```

## 3.2 FNN、CNN、RNN 对比

|模型|中文|核心结构|适合任务|易错点|
|---|---|---|---|---|
|FNN|前馈神经网络|信息单向传播|表格分类、回归|没有记忆能力|
|CNN|卷积神经网络|卷积 + 池化|图像、视频帧|核心不是循环，是卷积|
|RNN|循环神经网络|隐藏状态循环传递|文本、语音、时间序列|有梯度消失、长依赖问题|

## 3.3 CNN 为什么适合图像

CNN 有两个核心特征：

```text
局部连接
权重共享
```

图像中的局部区域有意义，比如边缘、纹理、角点、轮廓。卷积核可以在图片上滑动，提取局部特征。

CNN 处理图像的大致过程：

```text
底层：边缘、线条
中层：纹理、形状
高层：物体部件
最终：识别完整对象
```

## 3.4 梯度下降

梯度下降目标：

```text
最小化损失函数。
```

损失函数衡量：

```text
预测值和真实值之间的差距。
```

梯度下降过程：

```text
计算当前参数下的损失
    ↓
计算梯度
    ↓
沿负梯度方向更新参数
    ↓
重复直到损失较小
```

学习率：

```text
控制每次参数更新的步长。
```

学习率太大：

```text
震荡，甚至发散。
```

学习率太小：

```text
收敛很慢。
```

## 3.5 反向传播

反向传播作用：

```text
高效计算神经网络中每一层参数的梯度。
```

使用的数学原理：

```text
链式法则。
```

二者关系：

```text
反向传播负责算梯度；
梯度下降负责用梯度更新参数。
```

## 3.6 简答模板：梯度下降和反向传播

```text
梯度下降是一种优化算法，用于沿损失函数负梯度方向更新模型参数，使预测误差逐渐减小。反向传播是一种高效计算神经网络梯度的方法，它利用链式法则从输出层向输入层逐层传播误差，计算每一层参数对损失函数的影响。简单来说，反向传播负责计算梯度，梯度下降负责利用梯度更新参数，二者共同构成神经网络训练的核心机制。
```

## 4.6 大题模板：AI 系统组成

```text

算法层提供模型学习规律的方法，包括 FNN、CNN、RNN、梯度下降和反向传播等。FNN 是基础前馈网络，CNN 适合图像任务，RNN 适合序列任务；反向传播负责计算梯度，梯度下降负责更新参数。

模型层是在数据和算法基础上训练得到的智能系统。Transformer 通过自注意力机制解决了 RNN 难以并行和长距离依赖弱的问题，成为 GPT、BERT、DeepSeek 等大模型的核心架构。大模型通常采用预训练和微调范式，先学习通用能力，再适配具体任务。

应用层则把模型能力转化为实际产品，例如 AIGC 可以生成文本、图像、音频和视频，AI Agent 可以感知环境、规划任务、调用工具并执行操作。因此，AI 系统不是单一模型，而是由数据、算法、模型和应用共同构成的完整技术链条。
```

# 5. 第五重点：数据清洗与数据标注

这是学姐特别提醒的内容。不要以为它只是实践小节，它可以和 AI 系统组成一起考。

## 5.1 数据清洗是什么

定义：

```text
数据清洗是识别并处理原始数据中的缺失、重复、噪声、异常、格式不统一等问题，使数据更加准确、一致和可用的过程。
```

## 5.2 数据清洗处理什么问题

|问题|例子|处理方法|
|---|---|---|
|缺失值|年龄为空|删除、均值填充、中位数填充、特殊值填充|
|重复值|同一订单出现两次|去重|
|噪声|数据录入误差|平滑、分箱、回归|
|异常值|年龄 999|异常检测、人工核查|
|格式不统一|2025/6/1 和 2025-06-01|标准化|
|编码混乱|中文乱码|编码统一|

## 5.3 数据清洗流程

```text
明确错误类型
    ↓
识别错误实例
    ↓
纠正错误
    ↓
干净数据回流
```

## 5.4 数据标注是什么

定义：

```text
数据标注是通过分类、画框、注释等方式，对图片、语音、文本等原始数据添加机器可以学习的标签，使其成为机器学习训练样本的过程。
```

简单说：

```text
数据标注 = 给原始数据加答案。
```

## 5.5 为什么标注重要

监督学习需要：

```text
输入 X + 标签 y
```

模型学习的是：

```text
X 到 y 的映射关系。
```

如果标签错了：

```text
模型就会学错。
```

## 5.6 三类标注

|类型|常见任务|
|---|---|
|图像标注|图像分类、矩形框、多边形、语义分割、实例分割、关键点|
|语音标注|语音转写、时间轴、说话人分离、情绪标注|
|文本标注|文本分类、情感标注、实体识别、关系抽取、意图识别|

## 5.7 图像标注重点

矩形框标注：

```text
用于目标检测，例如车辆、人、猫狗、交通标志。
```

要求：

```text
框贴合目标边缘
不能太大
不能太小
类别标签正确
不能漏标
不能重复标
```

LabelMe：

```text
开源图像标注工具，支持矩形框、多边形、关键点，输出 JSON。
```

YOLO：

```text
目标检测模型，训练需要图片 + 类别 + 目标框坐标。
```

## 5.8 文本标注重点

文本标注难点：

```text
实体类型模糊
实体边界模糊
上下文依赖
新实体和领域词不断出现
反讽和复杂情绪难判断
```

例子：

```text
巴黎是法国首都。
巴黎协定具有重要意义。
```

第一个“巴黎”是地名，第二个“巴黎协定”是协议名称，必须结合上下文判断。

## 5.9 数据清洗与标注大题模板

```text
数据清洗和数据标注是人工智能模型训练前的重要数据处理环节。原始数据往往存在缺失值、噪声、重复记录、异常值和格式不统一等问题，如果直接用于训练，会影响模型学习效果。因此需要先进行数据清洗，包括缺失值处理、重复数据去重、异常值检测、格式标准化等，使数据更加准确和一致。

数据标注是在清洗后的数据上添加机器可以学习的标签。例如，图像标注可以给车辆和行人画矩形框，文本标注可以标出实体、关系和情感，语音标注可以把音频转写为文字并标出时间轴。对于监督学习来说，模型依赖带标签样本学习输入和输出之间的关系，因此标注数据相当于给模型提供标准答案。

标注质量会直接影响模型效果。如果图像框偏移、类别错标、目标漏标，目标检测模型就会学习错误规律；如果文本实体边界不准，NLP 模型的实体识别能力会下降；如果语音时间轴错位，语音识别模型也会受到影响。因此，一个完整的数据生产流程应包括数据采集、数据清洗、制定标注规范、人工标注、质量检查、返工修正和标准格式导出。
```

# 8. 第八重点：AI Agent

## 8.1 Agent 是什么

定义：

```text
AI Agent 是能够感知环境、基于目标自主决策，并采取行动影响环境以完成任务的智能实体。
```

核心闭环：

```text
感知环境
    ↓
理解目标
    ↓
规划步骤
    ↓
调用工具
    ↓
执行动作
    ↓
观察反馈
    ↓
调整策略
```

## 8.2 Agent 和生成式 AI 的区别

|生成式 AI|AI Agent|
|---|---|
|生成内容|完成任务|
|一问一答|多轮执行|
|被动响应|可主动规划|
|通常不调用工具|工具调用是核心|
|适合写作、问答|适合复杂任务执行|

## 8.3 Agent 核心模块

|模块|作用|
|---|---|
|感知模块|获取和理解外部信息|
|认知决策模块|理解目标、推理、规划|
|行动模块|调用工具、API、数据库、代码执行器|
|记忆模块|保存上下文、知识、历史经验|
|学习适应模块|根据反馈优化行为|

## 8.4 CoT / ReAct / RAG / Function Calling / MCP

|概念|含义|重点|
|---|---|---|
|CoT|Chain of Thought，思维链|分步骤推理|
|ReAct|Reason + Act|Thought → Action → Observation|
|RAG|检索增强生成|用知识库降低幻觉|
|Function Calling|函数调用|模型生成函数名和参数|
|MCP|模型上下文协议|标准化连接工具、数据、服务|

## 8.5 Function Calling 流程

```text
定义工具
    ↓
LLM 判断是否需要调用
    ↓
生成函数名和参数
    ↓
外部程序执行
    ↓
结果返回 LLM
    ↓
LLM 生成最终回答
```

## 8.6 RAG 流程

```text
上传文档
    ↓
文本切分
    ↓
向量化
    ↓
存入向量数据库
    ↓
用户提问
    ↓
检索相关片段
    ↓
交给大模型生成答案
```

## 8.7 Agent 论述模板

```text
AI Agent 是能够感知环境、基于目标进行自主决策，并采取行动完成任务的智能系统。与普通生成式 AI 不同，生成式 AI 主要根据提示生成文本、图像、代码等内容，而 Agent 更强调目标导向和任务执行。它通常包括感知模块、认知决策模块、行动模块和记忆模块。

感知模块负责理解用户输入、文档、图像、语音或结构化数据；认知决策模块以大语言模型为核心，负责理解目标、推理和规划；行动模块通过 Function Calling、API、数据库或代码执行器完成实际操作；记忆模块通过短期记忆、长期记忆、向量数据库和 RAG 提供上下文和外部知识支持。ReAct 使 Agent 能在 Thought、Action、Observation 的循环中边思考边行动，而 MCP 则提供更标准化的工具和数据连接方式。因此，AI Agent 代表 AI 从“生成内容”向“完成任务”的升级。
```


二者都指“猫”，但机器看到的是完全不同的数据形式。

# 13. 最终冲刺版：必须背的 6 段

## 1. AI 发展史

```text
人工智能的发展可以看作从规则驱动到数据驱动，再到大模型驱动和智能体驱动的过程。早期符号主义强调规则和逻辑推理，专家系统采用知识库和推理机解决特定领域问题，但存在知识获取困难和泛化能力弱的问题。随后机器学习和深度学习推动模型从数据中自动学习规律，CNN、RNN、反向传播和 GPU 加速提升了感知能力。2017 年 Transformer 提出后，自注意力机制解决了长距离依赖和并行训练问题，GPT、BERT、DeepSeek 等模型推动 AI 进入大模型时代。进一步地，AIGC 让 AI 能生成内容，AI Agent 则让 AI 从回答问题走向执行任务。
```

## 2. Transformer vs RNN

```text
RNN 通过循环结构按时间步处理序列，当前状态依赖当前输入和前一时刻隐藏状态，因此适合文本、语音和时间序列任务。但 RNN 难以并行训练，长序列中容易出现梯度消失或爆炸，长距离依赖建模能力较弱。Transformer 使用自注意力机制，使序列中每个位置都能直接关注其他位置，通过 Q、K、V 计算注意力权重并聚合上下文信息。多头注意力进一步增强模型对不同语义关系的捕捉能力。因此 Transformer 相比 RNN 的最大优势是支持并行训练和更好建模长距离依赖，适合大规模数据和大参数模型训练。
```

## 3. 神经网络训练

```text
神经网络是由输入层、隐藏层和输出层构成的可训练函数。FNN 信息单向传播，适合一般分类和回归；CNN 通过卷积和池化提取图像局部特征，适合图像任务；RNN 通过循环隐藏状态处理序列数据，适合文本、语音和时间序列。训练神经网络时，损失函数衡量预测值与真实值的差距，反向传播利用链式法则计算各层参数梯度，梯度下降则沿负梯度方向更新参数，使损失逐渐减小。
```

## 4. 数据清洗与标注

```text
数据清洗是对原始数据中的缺失值、噪声、重复记录、异常值和格式不统一等问题进行处理，使数据更加准确、一致和可用。数据标注是通过分类、画框、注释等方式给图片、语音、文本等数据添加机器可学习的标签。监督学习依赖带标签样本学习输入和输出之间的关系，因此标注质量直接影响模型效果。如果标签错误、漏标或边界不准，模型会学习错误规律，导致识别偏差和泛化能力下降。
```

## 5. AI Agent

```text
AI Agent 是能够感知环境、基于目标自主决策，并采取行动影响环境以完成任务的智能实体。它通常包括感知模块、认知决策模块、行动模块和记忆模块。相比普通生成式 AI，Agent 不只是生成内容，而是能够围绕目标进行任务分解、调用工具、执行操作并根据反馈调整策略。RAG、Function Calling、ReAct、MCP 和向量数据库等技术共同支撑现代 Agent 的实现。
```

## 6. AIGC

```text
AIGC 是 AI Generated Content，即人工智能生成内容，指利用人工智能生成文本、图像、音频、视频等内容。它在新闻写作、广告设计、电商商品图、语音克隆、数字人、游戏 NPC、影视制作等领域具有降本增效和创新体验的价值。但 AIGC 也存在版权争议、虚假信息、深度伪造、声音冒用和隐私泄露等风险，因此需要授权、标注、审核和合规管理。
```

## 1. 什么是开源

核心定义：

```text
开源是指软件源代码在许可证允许的条件下向公众开放，用户可以查看、使用、复制、修改和分发代码。
```

必须区分：

|概念|正确理解|易错点|
|---|---|---|
|开源|有许可证约束的源代码开放|不是随便用|
|免费|不收费|免费不一定开源|
|代码公开|能看到代码|公开不等于开源|

高频判断：

```text
代码上传到 GitHub/Gitee 就是开源。——错
开源软件一定免费。——错
免费软件一定开源。——错
开源软件可以商业化。——对
没有许可证的公开代码可以随便用。——错
```

## 2. 开源许可证

许可证作用：

```text
明确别人能不能复制、修改、分发、商用你的代码，以及需要承担什么义务。
```

三类最常考：

|许可证|类型|特点|
|---|---|---|
|MIT|宽松式|最简单、最宽松、允许闭源商用，保留版权声明即可|
|Apache 2.0|宽松式|允许商用，增加专利授权和专利保护，适合企业级项目|
|GPL|强 Copyleft|衍生作品分发时也必须 GPL 开源，有“传染性”|

记忆法：

```text
MIT：最宽松
Apache 2.0：宽松 + 专利保护
GPL：强制开源传递
```

# 四、第三大模块：神经网络基础

这部分是 AI 理论基础，选择题和论述题都可能考。

## 1. FNN / CNN / RNN

|模型|全称|核心特点|适合任务|
|---|---|---|---|
|FNN|前馈神经网络|信息单向传播，无记忆|表格数据、普通分类、回归|
|CNN|卷积神经网络|卷积、池化、局部连接、权重共享|图像、视频帧、医学影像|
|RNN|循环神经网络|有隐藏状态，能处理序列|文本、语音、时间序列|

重点解释：

```text
FNN：基础网络，信息从输入层到输出层单向传播
CNN：通过卷积核提取图像局部特征，适合空间结构数据
RNN：通过循环结构保存历史信息，适合序列数据，但有梯度消失和长距离依赖问题
```

## 2. 梯度下降与反向传播

损失函数：

```text
衡量模型预测值和真实标签之间的差距。
```

梯度下降：

```text
沿损失函数负梯度方向更新参数，使损失逐渐减小。
```

反向传播：

```text
利用链式法则，从输出层向输入层逐层计算每个参数的梯度。
```

二者关系：

```text
反向传播负责算梯度；
梯度下降负责用梯度更新参数。
```

考试答法：

```text
梯度下降是一种优化算法，通过沿损失函数负梯度方向更新参数，使模型误差逐渐减小。反向传播是一种高效计算神经网络梯度的方法，它利用链式法则从输出层向输入层逐层传播误差，计算各层参数对损失的影响。反向传播负责计算梯度，梯度下降负责利用梯度更新参数。
```

# 五、第四大模块：Transformer 与大语言模型

这是最高优先级。大题最可能围绕这里。

## 1. RNN 的问题

RNN 按时间顺序处理序列：

```text
x1 → h1 → x2 → h2 → x3 → h3 ...
```

问题：

```text
难以并行训练
长距离信息需要一步步传递
容易梯度消失或爆炸
长距离依赖建模弱
```

## 2. Transformer 的核心

Transformer 核心是：

```text
自注意力机制 Self-Attention
```

自注意力让每个 token 都能直接关注其他 token。

例如：

```text
小明把书放进书包，因为它很重。
```

模型要判断“它”指什么，需要关注前文中“书”“书包”等词。

## 3. Q、K、V

|名称|含义|直观解释|
|---|---|---|
|Query|查询向量|我想找什么信息|
|Key|键向量|我能提供什么索引|
|Value|值向量|我真正携带的信息|

流程：

```text
Query 和 Key 计算匹配程度
    ↓
得到注意力权重
    ↓
用权重对 Value 加权求和
    ↓
得到融合上下文的新表示
```

## 4. 多头注意力

多头注意力：

```text
多个注意力头在不同表示子空间中并行学习关系。
```

作用：

```text
有的头关注语法
有的头关注指代
有的头关注局部搭配
有的头关注主题关系
```

## 5. Transformer 相比 RNN 的优势

必须背：

```text
第一，Transformer 支持并行训练，不需要像 RNN 一样逐词串行处理。
第二，自注意力机制让任意两个 token 可以直接交互，更容易捕捉长距离依赖。
第三，多头注意力能从多个角度捕捉复杂语义关系。
第四，Transformer 更适合大规模数据和大参数模型训练，因此成为 GPT、BERT、DeepSeek、CLIP、ViT 等模型基础。
```

## 6. 大题模板：Transformer vs RNN

```text
RNN 通过循环结构处理序列，当前隐藏状态依赖当前输入和前一时刻隐藏状态，因此适合文本、语音和时间序列任务。但 RNN 必须按时间顺序串行计算，难以并行训练；当序列较长时，信息需要经过很多时间步传递，容易出现梯度消失或梯度爆炸，导致长距离依赖建模能力较弱。

Transformer 的核心是自注意力机制。自注意力允许序列中每个位置直接关注其他所有位置，通过 Query、Key、Value 计算注意力权重，并对 Value 加权求和，从而得到融合上下文的信息表示。多头注意力进一步让模型在多个表示子空间中同时捕捉不同类型的关系，如语法关系、指代关系和主题关系。

因此，Transformer 相比 RNN 的最大优势在于并行训练能力和长距离依赖建模能力。它不需要逐步传递隐藏状态，可以让任意两个 token 直接交互，更适合大规模数据和大参数模型训练，成为 GPT、BERT、DeepSeek 等现代大模型的基础架构。
```

# 六、第五大模块：大模型开源生态

这部分适合简答题、论述题。

## 1. 预训练 + 微调

预训练：

```text
在海量无标签数据上进行自监督学习，学习通用语言规律和世界知识。
```

微调：

```text
在特定任务数据上继续训练，使基础模型适配具体场景。
```

意义：

```text
基础模型训练成本高，但可以被复用；
微调成本较低，适合垂直领域应用；
开源基础模型降低了普通开发者和中小团队参与门槛。
```

## 2. LoRA / QLoRA / PEFT

PEFT：

```text
参数高效微调，只训练少量新增参数或小模块。
```

LoRA：

```text
冻结原模型参数，只训练两个低秩小矩阵 A 和 B，用低秩矩阵近似参数更新 ΔW。
```

QLoRA：

```text
LoRA + 4 bit 量化，进一步降低显存占用。
```

对比：

|方法|核心|作用|
|---|---|---|
|全参数微调|更新全部参数|成本高、显存高|
|LoRA|只训练低秩小矩阵|降低训练参数量|
|QLoRA|低秩微调 + 量化|进一步降低显存|

## 3. vLLM / TensorRT-LLM

推理：

```text
模型训练完成后，接收用户输入并生成输出的过程。
```

推理优化关注：

```text
延迟 latency
吞吐 throughput
显存利用率
并发能力
```

vLLM：

```text
通过 PagedAttention 高效管理 KV Cache，减少显存浪费，提高并发吞吐。
```

TensorRT-LLM：

```text
NVIDIA 推理优化框架，利用硬件级优化、低精度推理、算子融合等提升性能。
```

## 4. Hugging Face / ModelScope

Hugging Face 三大核心：

|模块|作用|
|---|---|
|Model Hub|模型共享|
|Dataset Hub|数据集共享|
|Spaces|AI 应用部署|

ModelScope：

```text
中国本土 AI 开源平台，强调中文语境、本土模型、模型即服务、一站式训练和部署。
```

考试答法：

```text
Hugging Face 是全球 AI 开源社区，核心包括 Model Hub、Dataset Hub 和 Spaces，分别支持模型共享、数据共享和应用部署。ModelScope 是中国本土 AI 开源平台，更强调中文语境、本土行业场景、模型即服务和一站式开发流程。二者共同推动了模型、数据、工具和应用的开源生态。
```

# 七、第六大模块：多模态 AI

## 1. 什么是多模态

多模态 AI：

```text
能够同时处理和关联文本、图像、音频、视频等多种数据类型的人工智能系统。
```

常见模态：

```text
文本
图像
音频
视频
传感器数据
```

## 2. 多模态难点

核心不是“数据多”，而是：

```text
如何融合？
如何表示？
如何对齐？
```

关键词：

|概念|含义|
|---|---|
|模态鸿沟|不同模态原始表示差异大，语义空间不一致|
|模态融合|把多种模态信息结合起来|
|跨模态对齐|让图片、文本等不同模态指向同一语义空间|
|统一表征|用统一模型处理多模态信息流|

## 3. CLIP

CLIP：

```text
Contrastive Language-Image Pre-training
```

核心：

```text
用图像编码器和文本编码器分别编码图片和文本；
把二者投影到共享语义空间；
匹配图文对拉近，不匹配图文对推远。
```

作用：

```text
图文检索
零样本分类
Stable Diffusion 文本理解
视觉语言模型基础组件
```

考试答法：

```text
CLIP 是一种图文对齐模型，由图像编码器和文本编码器组成双塔结构。它通过对比学习把图像和文本映射到共享语义空间，使匹配的图文距离更近，不匹配的图文距离更远，因此具备零样本分类和跨模态检索能力。
```

## 4. Stable Diffusion / ControlNet

Stable Diffusion：

```text
开源文生图扩散模型，可根据文本提示词生成图片。
```

正向提示词：

```text
希望图片中出现什么。
```

负向提示词：

```text
不希望图片中出现什么。
```

ControlNet：

```text
把线稿、姿态、边缘、深度图等结构信息作为控制条件，使生成图片符合指定构图。
```

# 九、第八大模块：AI Agent

## 1. AI Agent 定义

AI Agent：

```text
能够感知环境、基于目标进行自主决策，并采取行动影响环境以完成任务的智能实体。
```

核心闭环：

```text
感知环境 → 思考决策 → 执行动作 → 获取反馈 → 调整策略
```

## 2. Agent vs 生成式 AI

|生成式 AI|AI Agent|
|---|---|
|生成内容|完成任务|
|一问一答|多轮执行|
|通常不主动|可自主规划|
|不一定调用工具|工具调用是核心能力|
|适合写作、问答|适合复杂任务执行|

## 3. Agent 核心模块

```text
感知模块
认知决策模块
行动模块
记忆模块
学习适应机制
```

## 4. CoT / ReAct / RAG / Function Calling / MCP

|概念|含义|
|---|---|
|CoT|思维链，增强内部推理|
|ReAct|Thought → Action → Observation，推理与行动结合|
|RAG|检索增强生成，用外部知识降低幻觉|
|Function Calling|模型生成函数名和参数，由外部程序执行|
|MCP|标准化连接模型与工具、数据、服务的协议|

Function Calling 流程：

```text
定义工具
    ↓
LLM 判断是否需要调用
    ↓
生成函数名和参数
    ↓
外部程序执行
    ↓
结果返回模型
    ↓
模型生成最终回答
```

RAG 流程：

```text
上传文档
    ↓
文本切分
    ↓
向量化
    ↓
存入向量数据库
    ↓
用户提问
    ↓
检索相关片段
    ↓
交给 LLM 生成答案
```

# 十、第九大模块：数据清洗与数据标注

这是学姐特别提到的点，要重视。

## 1. 数据清洗

目的：

```text
处理原始数据中的缺失、重复、噪声、异常和格式不一致问题。
```

方法：

|问题|方法|
|---|---|
|缺失值|删除、均值填充、中位数填充、特殊值填充|
|噪声|平滑、分箱、回归、异常检测|
|重复数据|匹配、去重|
|格式不统一|标准化、规范化|

流程：

```text
明确错误类型
    ↓
识别错误实例
    ↓
纠正错误
    ↓
干净数据回流
```

## 2. 数据标注

定义：

```text
通过分类、画框、注释等方式，对图片、语音、文本等数据进行处理，使其成为机器学习训练样本。
```

核心理解：

```text
数据标注 = 给原始数据加机器可以学习的“答案”。
```

## 3. 标注类型

|类型|任务|
|---|---|
|图像标注|分类、矩形框、多边形、分割、关键点|
|语音标注|转写、时间轴、说话人、情绪|
|文本标注|实体、关系、情感、意图|

## 4. 标注质量

标注质量影响模型效果：

```text
标签错了，模型就会学错。
```

常见问题：

```text
漏标
错标
重复标
边界不准
类别混淆
文本实体边界模糊
情感反讽误判
语音时间轴错位
```

## 5. LabelMe / YOLO

LabelMe：

```text
开源图像标注工具，支持矩形框、多边形、关键点，输出 JSON。
```

YOLO：

```text
目标检测模型，训练需要图片 + 类别 + 目标框坐标。
```

实践链路：

```text
采集图片
    ↓
数据清洗
    ↓
LabelMe / LabelImg 标注
    ↓
质量检查
    ↓
转换为 YOLO / COCO 格式
    ↓
训练目标检测模型
```

# 十二、最后冲刺背诵版

## 必背 1：AI 发展史

```text
人工智能经历了从规则驱动到数据驱动，再到大模型驱动和智能体驱动的发展过程。早期符号主义强调规则和逻辑推理，专家系统采用知识库和推理机解决特定领域问题，但存在知识获取困难和泛化能力弱的问题。随后机器学习和深度学习推动模型从数据中自动学习规律，CNN、RNN、反向传播和 GPU 加速提升了感知能力。2017 年 Transformer 提出后，自注意力机制解决了长距离依赖和并行训练问题，GPT、BERT、DeepSeek 等模型推动 AI 进入大模型时代。进一步地，AIGC 让 AI 能生成内容，AI Agent 则让 AI 从回答问题走向执行任务。
```

## 必背 2：Transformer vs RNN

```text
RNN 通过循环结构按时间步处理序列，当前状态依赖当前输入和前一时刻隐藏状态，因此适合文本、语音和时间序列任务。但 RNN 难以并行训练，长序列中容易出现梯度消失或爆炸，长距离依赖建模能力较弱。Transformer 使用自注意力机制，使序列中每个位置都能直接关注其他位置，通过 Q、K、V 计算注意力权重并聚合上下文信息。多头注意力进一步增强模型对不同语义关系的捕捉能力。因此 Transformer 相比 RNN 的最大优势是支持并行训练和更好建模长距离依赖，适合大规模数据和大参数模型训练。
```

## 必背 3：数据清洗与标注

```text
数据清洗是对原始数据中的缺失值、噪声、重复记录、异常值和格式不统一等问题进行处理，使数据更加准确、一致和可用。数据标注是通过分类、画框、注释等方式给图片、语音、文本等数据添加机器可学习的标签。监督学习依赖带标签样本学习输入和输出之间的关系，因此标注质量直接影响模型效果。如果标签错误、漏标或边界不准，模型会学习错误规律，导致识别偏差和泛化能力下降。
```

## 必背 4：AI Agent

```text
AI Agent 是能够感知环境、基于目标自主决策，并采取行动影响环境以完成任务的智能实体。它通常包括感知模块、认知决策模块、行动模块和记忆模块。相比普通生成式 AI，Agent 不只是生成内容，而是能够围绕目标进行任务分解、调用工具、执行操作并根据反馈调整策略。RAG、Function Calling、ReAct、MCP 和向量数据库等技术共同支撑现代 Agent 的实现。
```

## 必背 5：AIGC

```text
AIGC 是 AI Generated Content，即人工智能生成内容，指利用人工智能生成文本、图像、音频、视频等内容。它在新闻写作、广告设计、电商商品图、语音克隆、数字人、游戏 NPC、影视制作等领域具有降本增效和创新体验的价值。但 AIGC 也存在版权争议、虚假信息、深度伪造、声音冒用和隐私泄露等风险，因此需要授权、标注、审核和合规管理。
```
