### 数据库系统的四大组件
- **数据 (Data):** 指实际存储在数据库中的内容 。文稿强调了数据是**集成的（integrated）** 和**共享的（shared）** 。
    
- **硬件 (Hardware):** 包括用于存储数据的磁盘 和用于运行DBMS软件的处理器与内存 。
    
- **软件 (Software):** 即数据库管理系统（DBMS） ，并列举了Oracle, DB2, SQL Server, Access, MySQL等常见的商业和开源DBMS产品 。
    
- **用户 (Users):** 分为三类：
    
    1. **应用程序员 (Application Programmers)** : 编写与数据库交互的应用程序。他们的工作包括编码、创建测试环境、SQL性能分析和打包部署等 。
        
    2. **最终用户 (End User)** : 通过应用程序或查询工具与数据库交互，以完成他们的业务任务，如运行应用、修改数据、生成报表和编写查询 。
        
    3. **数据/数据库管理员 (DA/DBA)**:
        
        - **DA (Data Administrator)** : 偏向于业务和管理层面，负责决定数据库中应存储哪些数据 ，并制定数据相关的政策，如安全策略 。
            
        - **DBA (Database Administrator)** : 偏向于技术实现层面，负责执行DA的决策 ，包括创建数据库、实施技术控制 、权限授予、性能监控、备份恢复等
- 这种四分法是理解数据库系统生态的经典模型。特别需要强调DA和DBA的区别：DA关注“**数据的商业价值和战略**”（What & Why），而DBA关注“**数据的技术实现和日常运维**”（How）。在一个大型企业中，这可能是两个不同的团队。一个好的DBA不仅要懂技术，还要理解业务需求，以便更好地进行数据库设计和性能优化。
    -想象一个电商平台。**数据**是用户表、商品表、订单表等。**硬件**是承载网站的服务器集群和存储阵列。**软件**是MySQL或PostgreSQL数据库。**用户**则包括：编写网站后端逻辑的**程序员**，在网站上购物的**最终用户**，决定需要收集哪些用户行为数据以供分析的**数据管理员(DA)**，以及负责保证MySQL数据库7x24小时高可用、高性能运行的**数据库管理员(DBA)**。这四个部分缺一不可，共同构成了一个完整的电商数据库系统。


- - DBA可以使用`GRANT`和`REVOKE`命令来精确地授予或撤销用户对特定表的操作权限 。此外，还可以通过创建
        
    **视图（View）**，让不同的用户只能看到他们被允许看到的数据子集，这既保证了安全，也简化了用户的操作 。
        * **视图（View）**是一个虚拟表，其内容由查询定义。视图不存储实际数据，数据仍然存储在基表中。它像一个窗口，用户通过这个窗口只能看到一部分数据。例如，可以为薪资部门的员工创建一个视图，只包含员工的姓名和薪水信息，而隐藏其他如家庭住址等隐私信息

### 数据模型与SQL语言

#### **实体-关系模型 (E-R Model)：概念结构的定义工具**

**E-R模型的定位与功能**： E-R模型是一种在**数据库概念设计阶段**使用的高层数据模型 。它的主要功能是将用户的业务需求，转化为一个独立于任何具体数据库软件（如MySQL或Oracle）的、纯粹的概念结构表示 。

**E-R模型的核心构成要素**：

- **实体 (Entity)**：
    
    - **定义**：指在目标系统中，需要被识别并为其存储信息的**对象类别** 。每个实体都代表了一组具有相同属性的、可区分的个体。
        
    - **功能解读**：在幻灯片248的示例中，`customer` 和 `account` 被定义为实体 。这是因为在银行这个业务系统中，“客户”和“账户”是系统需要管理和追踪的核心对象。系统需要存储每一个具体客户的信息和每一个具体账户的信息。
        
- **属性 (Attribute)**：
    
    - **定义**：指实体所具有的、需要被记录的**某个特定属性或特征** 。
        
    - **功能解读**：对于 `customer` 这个实体，系统需要记录其 `customer-id`（客户ID）、`customer-name`（客户名）、`customer-street`（街道）和 `customer-city`（城市）等信息 。这些信息就是`customer` 实体的属性。属性是对实体特征的具体化描述。
        
- **联系 (Relationship)**：
    
    - **定义**：指两个或多个实体类型之间的**相互关联** 。
        
    - **功能解读**：`customer` 和 `account` 实体之间并非毫无关联。一个客户可以在银行拥有账户并进行存款。`depositor` 这个联系就形式化地定义了这种业务关联 。“客户”通过“储户”这个联系与“账户”发生交互。一个联系可以有自己的属性，并且它明确了参与实体间的数量对应关系（例如，一个客户可以拥有多个账户，一个账户也可以被多个客户共同拥有）。
        

**E-R模型的工作流程**： 数据库设计的第一步，就是通过与业务人员沟通，识别出系统中的所有实体、实体的属性以及实体间的联系，并使用E-R图将其可视化。这份E-R图就是数据库的概念蓝图。设计完成后，E-R模型可以按照一套既定规则，被系统地**转换为关系模型** ，从而进入数据库的逻辑设计阶段。


#### **关系模型 (Relational Model)：逻辑结构的实现工具**

**关系模型的定位与功能**： 关系模型是一种**数据库逻辑设计阶段**使用的模型，它提供了可以被关系型数据库管理系统（RDBMS）直接理解和实现的逻辑结构。它的核心功能是将所有数据都组织成**二维表（Table）的集合** 。

**关系模型的核心构成要素**：

- **表 (Table) / 关系 (Relation)**：
    
    - **定义**：关系模型中的基本数据组织单位。它是一个由行和列组成的二维结构，用来存储关于某一特定实体类型的所有记录 。
        
- **行 (Row) / 元组 (Tuple)**：
    
    - **定义**：表中的一条水平记录，代表一个**具体的实体实例** 。例如，在
        
        `EMPLOYEE`表中，一行数据就代表一个具体的员工。
        
- **列 (Column) / 属性 (Attribute)**：
    
    - **定义**：表中的一栏垂直数据，代表实体的一个**特定特征** 。该列中所有的数据都属于相同的数据类型（如文本、数字、日期） 。
        
- **字段 (Field) 与 值 (Value)**：
    
    - **定义**：字段是表中特定行和特定列的交叉单元，它包含一个具体的**数据值** 。
        

**关系模型如何实现“联系”**： 在关系模型中，E-R模型中的“联系”对象不再独立存在。实体间的联系是通过在表中设置**公共列**来实现的，最典型的实现方式就是**外键 (Foreign Key)**。

- **功能解读**：在`EMPLOYEE`表中有一个`WORKDEPT`列，在`DEPARTMENT`表中有一个`DEPTNO`列 。
    
    `DEPARTMENT`表的`DEPTNO`是唯一标识部门的主键。`EMPLOYEE`表中的`WORKDEPT`列的值被设计为必须是`DEPARTMENT`表中某个已存在的`DEPTNO`值。通过这种方式，`WORKDEPT`列就建立了从`EMPLOYEE`记录到`DEPARTMENT`记录的引用，从而在逻辑上实现了员工与部门之间的“属于”关系。

- **DDL (Data Definition Language)** : 用于定义数据库模式（schema）的语言 。
    
- **示例:** `CREATE TABLE account (...)` 就是一条DDL语句，它定义了一个名为`account`的表，包含两个列，并指定了它们的数据类型和主键约束。
    
- **数据字典 (Data Dictionary):** DDL语句执行后，表的定义信息（如表名、列名、数据类型、约束等），即**元数据（metadata，关于数据的数据）** ，会被存储在数据字典中 。数据字典是DBMS核心的一部分。幻灯片286-301展示了如何查询Oracle数据库的数据字典视图（`user_tables`, `user_tab_columns`）来获取数据库的元数据

- `CREATE TABLE ...`: 创建一个新的表。
    
- `ALTER TABLE ...`: 修改一个已存在表的结构，如增加列、修改数据类型。
    
- `DROP TABLE ...`: 删除一个表及其所有数据。 这些都是DDL的核心命令。

**DML (Data Manipulation Language)** : 用于访问和操作数据的语言，也称为查询语言 。SQL是目前使用最广泛的DML ，它是一种**非过程化**语言 ，用户只需说明需要什么数据，而不用说明如何获取。

- `SELECT ...`: 从表中检索数据 。
    
- `INSERT INTO ...`: 向表中插入新的数据行 。
    
- `UPDATE ...`: 修改表中已存在的数据 。
    
- `DELETE FROM ...`: 从表中删除数据行
