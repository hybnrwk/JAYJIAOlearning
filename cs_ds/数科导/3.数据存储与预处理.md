**数据获取 → 数据存储 → 数据诊断 → 数据清洗 → 数据转换 → 可分析/可建模数据集**

课件前后逻辑是：第八讲讲“数据在数据库中的存储”，包括关系型数据库、非关系型数据库、Python 与 MySQL、Python 与 MongoDB；第九讲和补充课件讲“数据预处理”，包括数据类型、描述性统计、DataFrame 基本操作、重复值、缺失值、离群值、特征编码、标准化、离散化等。

# 一、先理解：为什么需要“存储”和“预处理”

数据获取之后，不能马上建模。因为真实数据通常不是干净、规整、可直接输入模型的。课件用房价数据举例：有缺失的 `SalePrice`，有重复的 `Id=16`，有明显异常的 `YearBuilt=2230`，还有缺失的类别字段。课件把现实数据的脏问题概括为三类：**不完整**、**噪音**、**重复**；补充课件还强调了**不一致**，例如编码或名称差异。

所以“数据存储”和“数据预处理”解决的是两个不同但连续的问题：

**数据存储**解决的是：数据应该以什么结构保存，方便后续查询、更新、管理和分析。

**数据预处理**解决的是：数据中的缺失、重复、异常、类型不一致、尺度不一致、类别无法建模等问题应该怎么处理。

课件里有一句非常重要的思想：没有质量保证的数据，就没有高质量的分析结果；模型输入数据质量会直接影响建模效果，正式建模之前需要做数据清洗、转换和规约。

# 二、数据存储部分：CSV、MySQL、MongoDB分别解决什么问题

## 1. 文件存储：CSV、Excel、JSON

课件重点讲数据库，但应用上先要知道文件存储的定位。

CSV、Excel、JSON 适合**小规模、一次性、结构简单**的数据保存。

例如：

一张课程成绩表。

一次爬虫得到的电影列表。

一份处理后的统计表。

一批接口返回的原始 JSON。

文件存储的优点是简单、直观、容易提交作业；缺点是缺少索引、约束、事务、权限管理，多表关联和复杂查询也不方便。所以文件适合“交作业”和“临时分析”，不适合长期维护的数据系统。

应用判断：

如果数据只有一张表、字段稳定、数据量不大、只分析一次，用 CSV 就够。

如果数据有多张表、有主键、有重复插入风险、需要反复查询，就应该考虑数据库。

如果数据是嵌套 JSON、字段经常变化，可以先用 JSON 文件或 MongoDB 保存原始结构。

## 2. 关系型数据库：MySQL

课件说，关系型数据库基于**关系模型（relational model）**，用二维表存储数据；列代表字段，行代表记录；实体之间的联系通过表和表之间的关联关系体现，通常借助主键和外键实现。关系型数据库由多个表组成，典型产品包括 Oracle、MySQL、SQL Server、SQLite、DB2 等。

MySQL 适合保存**结构化数据（structured data）**。所谓结构化，就是每一列是什么类型、每一行代表什么对象、表和表之间怎样关联，事先比较明确。

例如：

学生表：学号、姓名、班级、专业。

课程表：课程号、课程名、教师、学分。

选课表：学号、课程号、成绩。

这三张表之间有清楚的关系：学生和课程通过选课表发生多对多关系。

关系型数据库的核心特点：

**表结构固定**：字段、类型、约束事先设计好。

**关系明确**：表与表之间可以用主键、外键关联。

**支持 SQL**：可以用统一查询语言进行查询、插入、更新、删除。

**事务能力强**：适合银行转账、订单支付、库存管理等对一致性要求高的场景。MySQL 官方文档说明，InnoDB 支持 ACID 模型；ACID 是事务型数据库的重要可靠性原则，用于保证异常情况下数据不被破坏、结果不被扭曲。 ([MySQL开发者专区](https://dev.mysql.com/doc/refman/8.4/en/mysql-acid.html?utm_source=chatgpt.com "17.2 InnoDB and the ACID Model"))

应用判断：

如果你的数据像“表格”，字段固定，关系明确，后续要频繁按条件查询，选 MySQL。

如果你要防止重复插入，可以设置主键或唯一索引。

如果你需要多张表联查，比如“查询每个学生选了哪些课”，适合 MySQL。

如果你要保证多个操作要么同时成功、要么同时失败，适合 MySQL 的事务机制。

不适合 MySQL 的情况：

字段经常变。

一条记录内部有复杂嵌套结构。

每条记录字段差异很大。

只是临时保存原始爬虫 JSON。

这时更适合 MongoDB 或 JSON 文件。

## 3. Python 与 MySQL：不是重点背代码，而是理解操作流程

课件讲 Python 与 MySQL 存储时，重点是 PyMySQL 模块、MySQL 数据库安装、登录、创建数据库、创建表、插入数据等。

从应用角度，Python 操作 MySQL 的完整流程是：

第一，连接数据库。

第二，创建数据库或选择已有数据库。

第三，创建表，设计字段类型、主键、外键、唯一约束。

第四，把采集到的数据插入表。

第五，查询、更新或删除数据。

第六，提交事务并关闭连接。

你真正要理解的是表设计，而不是某一行代码。

例如学生成绩数据可以这样拆：

`student` 表保存学生基本信息。

`course` 表保存课程基本信息。

`score` 表保存学生-课程-成绩关系。

这样设计的好处是避免重复。例如课程名不用在每个学生记录里重复存一遍，而是通过课程号关联。

## 4. 非关系型数据库：NoSQL

课件说，非关系型数据库是一类不以关系表格为基础结构的数据存储系统，设计初衷是解决传统关系型数据库在大数据、高并发、可扩展性方面的局限。NoSQL 不是某一种技术，而是一个统称，包括键值型、文档型、列族型、图数据库等。

常见类型：

**键值型（key-value）**：如 Redis。适合高速缓存、计数器、会话信息。

**文档型（document-oriented）**：如 MongoDB。适合 JSON 类数据、半结构化数据。

**列族型（column-family）**：如 HBase、Cassandra。适合大规模分布式存储。

**图数据库（graph database）**：如 Neo4j。适合社交网络、知识图谱、关系路径分析。

非关系型数据库的优点是结构灵活、扩展性强、适合半结构化数据和大规模场景；缺点是不同类型 NoSQL 的数据模型差异大，不能简单用传统 SQL 思维统一处理。

## 5. MongoDB：文档型数据库

课件说 MongoDB 是 NoSQL 中功能较丰富、比较像关系数据库的一类；它基于键值对，数据结构松散，类似 JSON，可以存储复杂数据类型，性能较高。 MongoDB 官方文档也说明，MongoDB 把数据记录存为 BSON 文档；BSON 是 JSON 文档的二进制表示，并支持更多数据类型。([MongoDB](https://www.mongodb.com/docs/manual/core/document/?utm_source=chatgpt.com "Documents - Database Manual - MongoDB Docs"))

MongoDB 中几个核心概念：

**数据库（database）**：一组集合的容器。

**集合（collection）**：类似关系型数据库中的表，但不强制每条记录字段完全一致。

**文档（document）**：类似一行记录，但可以是嵌套结构。

例如一个商品文档可以包含：

商品名、价格、品牌、标签列表、评论列表、规格字典、抓取时间。

这种结构放在 MySQL 里可能要拆成商品表、标签表、评论表、规格表；放在 MongoDB 里可以一条文档直接保存。

应用判断：

如果数据来自网页接口，原始返回就是 JSON，优先考虑 MongoDB 保存原始数据。

如果字段不固定，比如有的商品有颜色、有的有尺寸、有的有配置参数，MongoDB 更灵活。

如果一条记录内部有列表或嵌套对象，比如评论列表、标签列表、图片列表，MongoDB 更自然。

如果后续要做严格多表关联和事务，MySQL 更合适。

# 三、MySQL 和 MongoDB 的选择：应用角度总结

|问题|MySQL 更适合|MongoDB 更适合|
|---|---|---|
|数据结构|字段固定、表结构明确|字段灵活、结构经常变化|
|数据形态|表格、关系记录|JSON、嵌套文档、半结构化数据|
|关联关系|多表关联、主外键|单文档内聚合保存|
|查询方式|SQL 查询、JOIN|文档查询、按字段过滤|
|一致性要求|高，一般更适合事务系统|灵活，适合快速写入和扩展|
|典型应用|学生-课程-成绩、订单-用户-商品|爬虫原始数据、日志、评论、商品详情|
|课程用途|学会规范表结构和 SQL 思维|学会保存复杂网页/接口数据|

最实用的存储策略是：

**原始采集数据先保存为 JSON 或 MongoDB；清洗后形成结构化表，再保存为 CSV 或 MySQL。**

这样既保留了原始信息，又方便后续分析。

# 四、数据预处理的总框架

课件第九讲的目录分四块：

第一，数据的初步诊断与探索：数据类型、描述性统计。

第二，数据的基本操作：数据类型、数据转换、过滤、选择、合并、排序。

第三，重复数据、缺失数据与离群数据处理。

第四，常用数据转换方法：特征编码、标准化、离散化。

补充课件还加入了相似性度量，包括距离、欧式距离、曼哈顿距离、余弦相似度等。

所以这部分的逻辑可以整理成：

**先认识数据 → 再诊断质量 → 再清洗错误 → 再转换格式 → 最后生成适合分析或建模的数据。**

# 五、数据类型：先判断数据是什么形态

课件把数据类型分为三大类：

**表格数据**：关系记录、数据矩阵、向量、事务数据。

**图和网络数据**：万维网、社交网络、分子结构。

**多媒体数据**：文本、图像、视频、音频。

## 1. 表格数据

表格数据是课程最主要处理对象。它类似 Excel 表或数据库表。

一行代表一个样本，一列代表一个特征。

例如：

一行是一个学生，一列是年龄、成绩、专业。

一行是一个房屋，一列是面积、价格、建造年份。

一行是一笔交易，一列是金额、时间、用户、商品。

表格数据最适合 Pandas、MySQL、Excel、统计建模。

## 2. 图和网络数据

图数据关注“对象之间的连接关系”。

例如：

社交网络中，人是节点，好友关系是边。

网页网络中，网页是节点，超链接是边。

交通网络中，站点是节点，道路是边。

图数据适合做路径分析、社群发现、PageRank、知识图谱等。普通 DataFrame 可以存边表，但更复杂的图分析通常需要图数据库或图算法库。

## 3. 多媒体数据

文本、图像、音频、视频都属于多媒体数据。它们通常不能直接像表格一样分析，需要先转成特征。

例如：

文本要分词、向量化、提取关键词或情感分数。

图像要转成像素矩阵或深度特征。

音频要提取频谱、MFCC 等特征。

视频通常由图像帧、音频、字幕、元数据共同组成。

课程里主要讲表格数据预处理，文本和图像属于后续更专门的方向。

# 六、样本、特征、属性：建模前必须分清

课件说，数据集由样本构成，一个数据对象表示一个实体；样本也可以叫 examples、instances、data points、objects、tuples。数据对象由属性及其值构成；数据库行是数据对象，数据库列是属性。

也就是说：

**样本（sample）**：一行数据，代表一个研究对象。

**特征（feature）**：一列数据，描述样本的某个方面。

**属性值（value）**：某个样本在某个特征上的具体取值。

例如房价数据：

一行房屋记录是一个样本。

`LotArea`、`YearBuilt`、`GarageType` 是特征。

某套房子的 `LotArea=8450` 是属性值。

这一步非常重要，因为后续所有处理都依赖“行”和“列”的语义。如果一行代表的是“用户”，就不能随便把多条购买记录合并成多行用户；如果一行代表的是“交易”，同一个用户出现多次并不是重复。

# 七、连续型特征和离散型特征

课件把特征分为连续型和离散型。连续型特征可以在实数空间任意取值，例如温度、身高、长度、价格，通常用浮点型表示；离散型特征的值域为有限集或可列集，例如汽车品牌、NBA 球队、布尔型、分类型、定序型。

应用上可以进一步分成：

**连续数值型**：身高、价格、面积、温度。适合求均值、标准差、标准化、回归。

**离散数值型**：购买次数、评论数、学生人数。虽然是数字，但通常是计数。

**名义型类别**：城市、品牌、专业、颜色。类别之间没有大小顺序。

**定序型类别**：低/中/高，差/良/优，一级/二级/三级。类别之间有顺序。

**布尔型**：是否有电梯、是否通过、是否违约。通常可以转成 0/1。

注意：不要把所有“数字”都当数值特征。例如学号、课程号、身份证号虽然由数字组成，但它们是 ID，不应该参与均值、方差、距离计算。

# 八、描述性统计：先获得数据总体印象

课件说，数据的基本统计信息用于获得总体印象，帮助理解数据；主要包括中心趋势、离散程度和图形显示。常用度量包括均值、中位数、最大值、最小值、分位数、方差等。

## 1. 集中趋势：均值、中位数、众数

**均值（mean）**：反映平均水平，公式是 $\bar{x}=\frac{1}{n}\sum_{i=1}^n x_i$。

适合：分布比较对称、没有严重极端值的数据。

不适合：收入、房价、播放量等长尾数据，因为极端值会拉高均值。

**中位数（median）**：排序后位于中间的值。

适合：有偏态、有极端值的数据。

例如统计工资时，中位数往往比均值更能代表普通人的水平。

**众数（mode）**：出现频率最高的值。

适合：类别变量。

例如最常见的户型、最常见的城市、最多的课程类型。

## 2. 离散程度：分位数、IQR、方差、标准差

**分位数（quantile）**：例如 Q1 是 25% 分位数，Q3 是 75% 分位数。

**四分位距 IQR**：$IQR=Q3-Q1$，表示中间 50% 数据的跨度。

**方差和标准差**：衡量数据围绕均值的波动程度。

如果标准差很大，说明样本之间差异大；如果标准差很小，说明数据比较集中。

## 3. 箱线图：识别离群值和偏态

课件对盒图讲得很细：Q1 是下四分位数，Q3 是上四分位数，IQR 是 $Q3-Q1$；下边缘是 $Q1-1.5IQR$，上边缘是 $Q3+1.5IQR$；盒外点常被视为离群值。箱线图可以直观识别离群值，也能观察偏态和尾部情况。

应用判断：

如果你只想快速发现异常值，用箱线图很有效。

如果数据强烈右偏，例如收入、房价、播放量，箱线图会显示很多高端离群点。

这些点不一定都是错误，可能是极端但真实的数据，需要结合业务判断。

# 九、Pandas DataFrame：预处理的主要操作平台

课件说，Pandas 中 Series 是一维结构，带索引；DataFrame 是二维、大小可变的表格数据结构，类似电子表格或 SQL 表，每列可以是不同类型；DataFrame 由多个 Series 按列组成。 Pandas 官方文档也把 DataFrame 定义为二维带标签的数据结构，列可以具有不同类型，可以理解为电子表格、SQL 表或由 Series 组成的字典。([Pandas](https://pandas.pydata.org/docs/user_guide/dsintro.html?utm_source=chatgpt.com "Intro to data structures — pandas 3.0.2 documentation - PyData |"))

DataFrame 的作用是：把外部数据读进内存，在 Python 中进行清洗、筛选、合并、统计、转换。

## 1. 查看数据

课件列出常见查看方式：

`head()` 看前几行。

`tail()` 看后几行。

`shape` 看行列数。

`columns` 看列名。

`index` 看索引。

`describe()` 看摘要统计。

`dtypes` 看数据类型。

应用意义：

刚拿到数据时，先不要急着处理。先看行数、列数、字段类型、数值范围、缺失情况。

## 2. 数据类型转换

课件列出 Pandas 常见类型：整数型、浮点型、字符串型、布尔型、日期时间型、分类型；也讲到可以用 `astype()` 或 `to_datetime()` 等方式做类型转换。

应用判断：

金额、面积、评分应该是数值型。

日期应该转为 datetime。

有限类别可以转为 category。

布尔字段可以转成 0/1。

ID 虽然可能是数字，但通常应保留为字符串。

类型转换很重要，因为类型错了，后续统计会错。例如价格字段如果是字符串，就不能正确求均值；日期如果是字符串，就不能按年月分组。

## 3. 数据选择、过滤、排序

课件讲了按列选择、按行选择、`.loc[]` 和 `.iloc[]`，也讲了布尔条件过滤、多个条件过滤、`isin()`、`query()`；排序可以用 `sort_values()`、`sort_index()`、`nlargest()`、`nsmallest()`。

应用上：

选择是为了取需要的字段。

过滤是为了保留符合条件的样本。

排序是为了看最大、最小、最新、最早或排名。

例如：

筛选价格大于 1000 的商品。

筛选某个城市的数据。

找销售额最高的前 10 个门店。

按时间排序观察趋势。

## 4. 数据合并：merge、concat、join

课件讲到，Pandas 合并数据主要有 `merge()`、`concat()`、`join()`。`merge()` 按一列或多列的值合并；`concat()` 沿行或列连接多个 DataFrame；`join()` 通常按索引合并，并可指定 inner、outer、left、right 等方式。

应用判断：

**concat**：适合同结构数据纵向拼接。例如 1 月数据 + 2 月数据 + 3 月数据。

**merge**：适合按共同字段关联两张表。例如学生表和成绩表按学号合并。

**join**：适合按索引合并。例如两个以日期为索引的时间序列表。

注意：合并后一定检查行数。如果合并后行数突然变多，可能发生了一对多或多对多合并；这不一定错，但要确认是否符合样本定义。

# 十、重复数据处理

课件对重复数据讲得很实用。重复数据的原因包括数据采集错误、数据合并问题、手动输入错误、系统逻辑缺陷；具体来源包括多次采集同一条数据、不同来源合并时没有去重、人工重复输入、系统未设置主键或唯一约束。重复数据会导致平均值/总和被放大、模型训练权重失衡、分组汇总不准确、图表重复。

## 1. 重复值检测

检测重复值的基本问题是：

**哪些字段共同决定“一条数据是否重复”？**

如果整行完全一样，可以按整行去重。

如果某个 ID 唯一，例如学号、订单号、视频 ID，可以按 ID 去重。

如果没有 ID，则可能要用多个字段组合判断，例如“标题 + 发布时间 + 作者”。

## 2. 重复值删除

课件中提到 `duplicated()` 可以检测重复行，`drop_duplicates()` 可以删除重复行；默认把完全相同的行视为重复，也可以用 `subset` 指定字段，还可以用 `keep='first'` 或 `keep='last'` 决定保留哪一个副本。

应用判断：

如果同一条记录被重复爬取，删除重复副本。

如果同一个用户多次购买，不应删除，因为这是不同交易。

如果同一商品每天采集一次价格，也不应简单去重，因为它是时间序列。

因此处理重复值前必须先定义“样本单位”。

# 十一、缺失值处理

课件讲缺失值处理方法包括删除法、数据特征填补法、随机填补法、Bootstrap 填补法、基于模型的填补。常见填补包括均值、中位数、众数、固定值、前向填充、后向填充、插值等。

补充课件还说明，缺失值可能来自数据采集过程、网络传输过程、数据整合过程。删除法既可以从样本角度删除，也可以从特征角度删除；它以减少数据为代价换取完整性，可能丢失隐藏信息。

## 1. 先判断缺失的含义

缺失值可能表示：

真的不知道。

不适用。

系统没采集到。

用户没填写。

合并时没有匹配上。

这几种含义完全不同。

例如：

车库类型缺失，可能表示“没有车库”，而不是“不知道车库类型”。

用户年龄缺失，通常表示“未填写”。

订单支付时间缺失，可能表示“未支付”。

所以不能一律均值填补。

## 2. 删除法

删除法包括删除样本和删除特征。

适合删除样本的情况：

缺失样本占比很小。

某些样本多个关键字段都缺失。

删除不会明显影响总体分布。

适合删除特征的情况：

某列缺失率很高。

该列对分析目标不重要。

该列缺失无法合理填补。

缺点是会丢失信息，尤其当缺失不是随机发生时，删除会造成偏差。

## 3. 均值填补

均值填补是用非缺失值的平均值替代缺失值，适合数值型变量。课件指出它简单快速、计算成本低，但会降低数据波动性、低估标准差，也不适合非正态分布数据或类别型变量。

适合：

缺失比例较低。

变量分布比较对称。

只是做简单分析或初步模型。

不适合：

强偏态变量。

有极端值变量。

缺失比例高。

变量间相关性很强。

## 4. 中位数填补

中位数不受极端值影响，适合偏态分布数据。课件也指出，中位数填补比均值更稳健，但同样会降低自然变异性，并忽略变量间关系。

适合：

收入、房价、播放量、销售额等长尾数据。

存在极端值的数据。

## 5. 众数填补

众数填补适合类别型变量。课件指出它简单易实现，但可能增加某个类别的权重，造成偏差，且无法反映实际数据多样性。

适合：

缺失比例不高的类别变量。

类别分布较稳定的字段。

如果缺失本身有含义，最好单独设为 `Unknown`、`None` 或 “未填写”。

## 6. 前向填充、后向填充、插值

这些方法适合有顺序的数据，尤其是时间序列。

例如每天温度缺失一天，可以用前后日期插值。

股票价格、传感器数据、日志指标可以用前向填充或插值。

但普通横截面数据，例如学生成绩表，不适合随便前向填充。

## 7. 基于模型的填补

当缺失字段很重要，而且与其他字段关系明显时，可以用模型预测缺失值。

例如用面积、地段、楼层预测缺失租金。

优点是利用变量间关系，缺点是复杂、容易引入模型误差，也要避免数据泄漏。

# 十二、离群值检测与处理

离群值是明显偏离大多数样本的数据点。课件讲了两类方法：基于统计的方法和基于近邻的方法。基于统计的方法包括上、下 $\alpha$ 分位点之外的值，以及盒图观察；基于近邻的方法包括 LOF 局部异常因子算法，其基本思想是比较点和邻域点的密度，密度越低，越可能是异常点。

## 1. 统计方法

常见统计方法包括：

分位数法。

箱线图 IQR 法。

Z-score 法。

业务规则法。

其中 IQR 法与课件盒图对应：小于 $Q1-1.5IQR$ 或大于 $Q3+1.5IQR$ 的值常被视作离群值。

适合：

单变量异常检测。

快速探索数据。

报告中解释异常值。

## 2. LOF 方法

LOF（Local Outlier Factor）是局部密度异常检测方法。它不是只看某个点的绝对大小，而是看它周围邻域的密度。如果某个点周围很稀疏，而邻居所在区域比较密集，它就更可能是异常点。课件也说明，LOF 值越接近 1，点越像正常点；大于 1 越多，越可能是异常点。

适合：

多维数据。

非线性分布。

异常点不是简单最大最小值的情况。

例如客户行为异常、交易异常、传感器异常。

## 3. 离群值处理方法

离群值不能一看到就删除。要先判断它是：

录入错误。

采集错误。

真实但极端。

研究对象本身。

处理方式包括：

修正错误值。

设为缺失再填补。

删除明显错误样本。

截尾或缩尾。

对数变换。

保留并打标记。

例如建造年份 2230 大概率是错误；但高收入用户、高播放量视频、高房价豪宅可能是真实极端值，不应简单删除。

# 十三、相似性度量：距离和余弦相似度

补充课件把相似性度量放在初步诊断与探索中。它的应用包括判断样本是否重复、两个商品是否相似、客户分群等。距离函数一般满足非负、自身距离为零、对称性、三角不等式。

## 1. 闵可夫斯基距离

闵可夫斯基距离是一个总框架。

当 $h=1$ 时，是曼哈顿距离。

当 $h=2$ 时，是欧式距离。

当 $h=\infty$ 时，是极大距离。

## 2. 曼哈顿距离

曼哈顿距离是各维度绝对差之和。

适合：

网格路径。

高维稀疏数据。

希望每个维度差异线性累加的情况。

## 3. 欧式距离

欧式距离是最常见的几何距离。

适合：

连续数值变量。

聚类分析。

KNN。

异常检测。

注意：使用欧式距离前通常需要标准化，否则尺度大的变量会支配距离。

## 4. 极大距离

极大距离只看各维度差异中的最大值。

适合：

只要某一维差异过大就认为整体差异大的情况。

## 5. 余弦相似度

余弦相似度关注向量方向，而不是绝对大小。补充课件强调，欧式距离衡量绝对距离，余弦相似度衡量方向差异；欧式距离适合用户行为指标绝对差异，余弦相似度适合用用户对内容评分来区分兴趣相似性。

应用例子：

文本向量相似度。

用户兴趣相似度。

商品标签相似度。

评分向量相似度。

# 十四、特征编码：把非数值特征转成模型能用的形式

课件说，模型输入的特征通常需要是数值型，所以要把非数值特征转换为数值特征，包括数字编码、One-Hot 编码、哑变量编码。

## 1. 数字编码

数字编码是把类别映射为数字。

例如收入水平：

贫困、低收入、小康、中等收入、富有。

编码为：

0、1、2、3、4。

适合：

本身有顺序的类别。

例如低/中/高，差/良/优。

缺点：

如果类别本来无顺序，例如颜色、城市、品牌，数字编码会人为引入大小关系。课件也明确指出，数字编码的缺点是会引入次序关系。

## 2. One-Hot 编码

One-Hot 编码把包含 $K$ 个取值的离散特征转换成 $K$ 个二元特征，每个取值对应一个 0/1 列。课件说，One-Hot 编码后不同原始特征之间拥有相同距离，对包含离散特征的回归模型和分类模型通常有帮助。 scikit-learn 官方文档也说明，`OneHotEncoder` 会把类别特征编码为 one-hot 数值数组，为每个类别创建二元列。([Scikit-learn](https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.OneHotEncoder.html?utm_source=chatgpt.com "OneHotEncoder"))

适合：

无序类别变量。

线性回归、逻辑回归、SVM、神经网络等模型。

类别数量不太多的字段。

不适合：

类别数量极多的字段，例如用户 ID、商品 ID、地址、课程号。否则会产生大量稀疏列。

## 3. 哑变量编码

哑变量编码与 One-Hot 很接近，通常会去掉一个基准类别，避免完全共线性。

例如性别有男、女，如果用两个变量 `is_male` 和 `is_female`，二者完全互补；在线性模型中通常只保留一个。

应用判断：

树模型对类别编码的要求相对不那么严格。

线性模型更需要谨慎编码。

无序类别不要直接数字编码。

高基数类别要考虑合并低频类别、统计编码或删除 ID。

# 十五、标准化：让不同尺度的数值可比较

课件总结了标准化方法：Z-score 标准化、Min-Max 标准化、小数定标标准化、Logistic 标准化。

标准化的核心问题是：不同变量量纲不同，数值范围不同，会影响模型。

例如：

年龄范围 0 到 100。

收入范围几千到几十万。

评分范围 0 到 5。

如果使用欧式距离，收入会压倒其他变量。

## 1. Z-score 标准化

公式是 $z=\frac{x-\mu}{\sigma}$。

含义是减去均值，再除以标准差，使变量均值接近 0，标准差接近 1。scikit-learn 官方文档也说明，`StandardScaler` 的作用是移除均值并缩放到单位方差，标准分数公式为 $z=(x-u)/s$。([Scikit-learn](https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.StandardScaler.html?utm_source=chatgpt.com "StandardScaler — scikit-learn 1.8.0 documentation"))

适合：

近似正态分布的数据。

线性模型。

KNN。

K-means。

PCA。

SVM。

神经网络。

注意：对离群值敏感。

## 2. Min-Max 标准化

把变量缩放到 $[0,1]$ 或其他指定区间。

公式是 $x'=\frac{x-x_{min}}{x_{max}-x_{min}}$。

适合：

需要固定范围的模型。

图像像素。

神经网络输入。

数据最小最大值比较稳定的场景。

缺点：如果有极端值，普通样本会被压缩到很小范围；课件也指出，当新数据加入导致最大值或最小值变化时，可能需要重新定义最大最小值，且存在离群值时效果较差。

## 3. 小数定标标准化

通过移动小数点位置缩放数据。

例如数据范围在 $[-3075,2187]$，可以除以 $10^4$，让数据落入较小范围。课件指出它适合比较分散、跨多个数量级的数据，但也容易受到离群值影响。

## 4. Logistic 标准化

利用 Logistic 函数把数据映射到 $[0,1]$。课件指出，它适合特征取值相对集中分布于 0 两侧的情况，但可能改变原始分布和取值关系，因此使用前要先分析原始分布。

## 5. 哪些模型需要标准化

强烈需要标准化：

KNN。

K-means。

SVM。

PCA。

逻辑回归。

线性回归带正则化。

神经网络。

距离型异常检测。

不太需要标准化：

决策树。

随机森林。

梯度提升树。

因为树模型主要根据阈值切分，不依赖变量尺度。

# 十六、离散化：把连续特征划分成区间

课件把离散化定义为：将连续型特征的取值范围划分为若干区间段（bin），然后用区间段代替落在该区间段的特征取值；区间之间的分割点叫切分点，切分出来的区间数叫元数。

例如年龄范围 $[0,150]$，用切分点 10、25、40、60 划分为 5 个区间。

离散化的目标是：

**在信息损失尽量少的前提下，尽可能减少取值个数。**

## 1. 为什么要离散化

课件说，某些算法对特征类型有要求，例如关联规则挖掘、ID3 决策树等；离散化有时也能提高算法精度。

应用上，离散化还有解释性价值：

年龄变成少年、青年、中年、老年。

收入变成低收入、中收入、高收入。

评分变成低分、中分、高分。

这样更适合做分组统计和业务解释。

## 2. 无监督离散化

无监督离散化不参考目标变量 $y$，只看特征本身分布。课件列出等距离散化、等频离散化、聚类离散化。

**等距离散化**：每个区间宽度相同。

适合数据分布比较均匀、区间解释重要的场景。

**等频离散化**：每个区间样本数量大致相同。

适合偏态分布，希望每组样本数量均衡的情况。

**聚类离散化**：根据数据聚类结果划分区间。

适合数据自然形成几个簇的情况。

## 3. 有监督离散化

有监督离散化利用目标变量 $y$，目的是让分箱更有助于预测目标。课件列出信息增益离散化、ChiMerge 离散化等。

**信息增益离散化**：选择能最大程度区分类别的切分点。

适合分类任务，和决策树思想接近。

**ChiMerge 离散化**：先把每个取值看作单独区间，再计算相邻区间的卡方统计量，把卡方值小或低于阈值的相邻区间合并，直到满足终止条件。

适合：

分类任务。

希望相邻区间在类别分布上有显著差异。

需要监督信息参与分箱的场景。

注意：有监督离散化不能在全数据上提前看测试集标签，否则会造成数据泄漏。

# 十七、数据可视化在预处理中的作用

课件作业要求中提到，选择爬取数据集后要进行恰当预处理，包括基本统计特征、缺失值处理、异常值检测及处理、数据编码、数据标准化，并尝试数据可视化。

可视化不是最后展示用的装饰，而是预处理诊断工具。

常用图形：

直方图：看分布是否偏态。

箱线图：看离群值。

散点图：看两个变量关系。

条形图：看类别频数。

热力图：看相关性或缺失模式。

折线图：看时间趋势。

应用判断：

如果你不知道用均值还是中位数填补，先看直方图和箱线图。

如果你不知道某个极端值该不该删，先看散点图和业务范围。

如果你不知道类别变量是否需要合并，先看类别频数条形图。

# 十八、从课件出发的完整操作流程

可以把这一章的方法组织成一套标准流程：

## 第一步：保存原始数据

先决定数据存储方式。

小规模表格：CSV。

结构化多表：MySQL。

半结构化 JSON：MongoDB。

原始数据不要直接覆盖。保留 raw 版本，后续清洗出 cleaned 版本。

## 第二步：读取到 DataFrame

用 Pandas 读取数据，形成 DataFrame。

查看：

行列数。

字段名。

数据类型。

前几行。

描述性统计。

缺失比例。

唯一值数量。

## 第三步：确认样本和特征

明确：

一行代表什么对象。

哪些列是 ID。

哪些列是数值型。

哪些列是类别型。

哪些列是日期型。

哪些列是目标变量。

## 第四步：描述性统计和可视化

对数值型看：

均值、中位数、最大值、最小值、分位数、标准差、箱线图。

对类别型看：

类别数量、众数、频数分布。

## 第五步：处理重复值

确定唯一键。

检测重复行。

决定保留 first、last，还是全部删除。

合并数据后尤其要检查重复。

## 第六步：处理缺失值

先分析缺失原因。

缺失少且随机：可删除或简单填补。

## 第七步：检测和处理离群值

用箱线图、分位数、Z-score、业务规则、LOF 等方法识别。

判断是错误还是极端真实值。

错误值修正或删除。

真实极端值可保留、标记、截尾或变换。

## 第八步：数据转换

类别变量编码。

数值变量标准化。

必要时离散化。

日期变量拆分成年、月、日、星期、是否周末等。

## 第九步：输出最终数据

保存：

原始数据。

清洗后数据。

建模特征数据。

同时记录处理规则，保证可复现。
