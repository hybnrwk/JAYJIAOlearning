# 一、数据可视化到底在做什么

数据可视化（data visualization）就是把数据转化成图形、图像或动画等形式，让人更直观看到数据中的结构、趋势、异常和关系。课件把它的作用概括为三点：**帮助理解数据、提升决策效率、增强沟通效果**。

在数据科学项目里，可视化不是最后做汇报时“美化一下”的步骤，而是贯穿整个流程：

**数据获取后**：用图检查数据是否异常。  
**数据预处理时**：用图判断缺失、离群、偏态、类别不均衡。  
**建模前**：用图观察变量与目标变量是否有关。  
**建模后**：用图展示预测效果、误差分布、特征重要性。  
**写报告时**：用图把复杂结果讲清楚。

所以这章真正要学的是：

> **根据数据类型和分析目的选择合适图表，并能解释图表表达了什么。**

# 二、画图前先问：我要回答什么问题？

选图表不能从“我想画散点图/柱状图”开始，而要从问题开始。

## 1. 看一个变量的分布：直方图、箱线图、密度图

问题形式：

“年龄主要集中在哪个范围？”

“房价是不是右偏分布？”

“播放量有没有极端值？”

“票价的中位数和异常值在哪里？”

适合图形：

**直方图（histogram）**：看数值变量的分布形状。  
**箱线图（boxplot）**：看中位数、四分位数、离群值。  
**密度图（density plot / KDE）**：看平滑后的分布趋势。  
**小提琴图（violin plot）**：同时看分布形状和箱线图信息。

课件中 Titanic 年龄分析就用了年龄直方图和年龄箱线图，分别观察年龄分布和离群值情况。 Seaborn 官方文档也说明，直方图是可视化分布最常见的方法，它把变量取值轴划分成若干区间，并用柱子的高度表示每个区间内的观测数量。([seaborn.pydata.org](https://seaborn.pydata.org/tutorial/distributions.html?utm_source=chatgpt.com "Visualizing distributions of data - Seaborn - PyData |"))

## 2. 看类别数量：柱状图、计数图

问题形式：

“不同舱位人数多少？”

“不同起航港人数多少？”

“钻石不同切工等级数量如何？”

“不同类别的样本是否均衡？”

适合图形：

**柱状图（bar chart）**：展示类别之间的数值大小。  
**计数图（countplot）**：统计每个类别出现次数。

课件中 Titanic 案例用 `countplot` 画所有乘客生还/遇难人数，也用它画男女比例、不同客舱人数、不同起航港人数。

这里要注意：柱状图看的是**数量**，但很多时候我们真正关心的是**比例或概率**。例如三等舱人数最多，不代表三等舱生还率最高。要比较“生还率”，必须计算每组中生还人数 / 该组总人数。

## 3. 看两个数值变量之间的关系：散点图

问题形式：

“花萼长度和花瓣长度有没有关系？”

“房屋面积越大，价格是否越高？”

“播放量和点赞数是否正相关？”

适合图形：

**散点图（scatter plot）**。

课件用鸢尾花数据画了 `sepal length` 与 `petal length` 的散点图，并用颜色和点型区分不同类别。这个图能直接看到不同类别在二维空间中的分布差异。

散点图最适合观察：

变量之间是否正相关、负相关。

是否存在明显分群。

是否存在离群点。

是否存在线性或非线性关系。

如果点太多发生重叠，可以考虑透明度 `alpha`、抽样、hexbin 六边形箱图或二维密度图。

## 4. 看数值变量随时间变化：折线图

问题形式：

“股票价格随时间怎么变化？”

“每日订单量是否上升？”

“模型 loss 是否下降？”

“评论数量是否随时间增加？”

适合图形：

**折线图（line chart）**。

折线图适合横轴有顺序，尤其是时间。它强调趋势，不适合横轴类别没有顺序的情况。

课件中 Matplotlib 基础部分用 `plt.plot()` 画线图，展示多组函数曲线；金融部分的 K 线图本质上也是时间序列可视化的一种，只不过它同时展示开盘价、收盘价、最高价和最低价。

## 5. 看类别变量与数值变量的关系：箱线图、小提琴图、点图

问题形式：

“不同星期的小费金额分布是否不同？”

“吸烟者和非吸烟者账单金额有什么差异？”

“不同企鹅物种的喙长度是否不同？”

适合图形：

**箱线图（boxplot）**：比较各组的中位数、四分位数、离群点。  
**小提琴图（violin plot）**：比较各组完整分布形状。  
**点图（pointplot）**：比较分组均值或估计值及其误差范围。

课件中用 tips 数据集画了 `day`、`total_bill`、`smoker` 的箱线图和小提琴图，用 penguins 数据集画了岛屿、喙长、物种之间的点图。 Seaborn 官方文档对 boxplot 的定义也强调，它用于显示定量数据在不同类别下的分布，盒体显示四分位数，须线显示其余分布，并可显示离群点。([seaborn.pydata.org](https://seaborn.pydata.org/generated/seaborn.boxplot.html?utm_source=chatgpt.com "seaborn.boxplot — seaborn 0.13.2 documentation - PyData |"))

## 6. 看变量之间的整体关系：热力图、pairplot

问题形式：

“哪些变量相关性强？”

“多个特征之间是否有聚类结构？”

“所有数值变量两两关系如何？”

适合图形：

**热力图（heatmap）**：常用于相关矩阵、混淆矩阵、缺失值模式。  
**成对图（pairplot）**：展示多个变量两两之间的散点图和单变量分布。

课件列出了 Seaborn 可以画的关系图、分类图、分布图、回归图、矩阵图等，其中热力图、pairplot、jointplot、lmplot 都属于更偏探索性数据分析的工具。

# 三、Matplotlib：最基础、最灵活的绘图库

课件说，Matplotlib 是 Python 中比较基础的绘图库，基于 NumPy 数组运算，可视化功能强，是 Python 中最基础的可视化工具之一。 官方文档中，Matplotlib 的基本结构是 **Figure + Axes**：Figure 是整张图的容器，Axes 是实际绘制数据的区域，Axes 中包含坐标轴、刻度、标签、标题等元素。([Matplotlib](https://matplotlib.org/stable/users/explain/quick_start.html?utm_source=chatgpt.com "Quick start guide — Matplotlib 3.10.9 documentation"))

## 1. 先理解 Figure 和 Axes

你可以把 Matplotlib 理解成：

**Figure = 画布**  
**Axes = 画布上的一个图表区域**  
**Axis = 坐标轴，如 x 轴、y 轴**  
**Title = 图标题**  
**Legend = 图例**  
**Tick = 刻度**  
**Grid = 网格线**

课件第 8 页列出 Matplotlib 图表的组成元素：Figure、Axes、Title、Legend、X/Y 轴标签、主刻度、次刻度、刻度标签、网格线等。

这部分最容易混淆的是：

**Axes 不是 Axis。**

Axes 是整个作图区域；Axis 是 x 轴或 y 轴。

## 2. Matplotlib 常见图形

课件列出 Matplotlib 中常用图形及函数：

`plt.plot()`：线图。  
`plt.scatter()`：散点图。  
`plt.bar()`：柱状图。  
`plt.hist()`：直方图。  
`plt.boxplot()`：箱线图。  
`plt.pie()`：饼图。  
`plt.contour()`：等高线图。  
`plt.imshow()`：热力图像。  
`plt.errorbar()`：误差棒图。  
`plt.hexbin()`：六边形箱图。  
`plt.polar()`：极坐标图。 Matplotlib 官方也提供了大量图表示例库，覆盖折线图、散点图、条形图、图像、等高线等多种可视化类型。([Matplotlib](https://matplotlib.org/stable/gallery/index.html?utm_source=chatgpt.com "Examples — Matplotlib 3.10.9 documentation"))

应用上可以这样记：

|分析目的|Matplotlib 图形|
|---|---|
|看趋势|折线图 `plot`|
|看两个变量关系|散点图 `scatter`|
|比较类别大小|柱状图 `bar`|
|看数值分布|直方图 `hist`|
|看离群值|箱线图 `boxplot`|
|看占比|饼图 `pie`|
|看二维强度|热力图 `imshow`|
|看高密度散点|六边形箱图 `hexbin`|
|看误差区间|误差棒图 `errorbar`|

## 3. Matplotlib 参数：不是背，而是知道在控制什么

课件中的代码示例设置了颜色、线型、线宽、点型、点大小、标题、坐标轴标签、图例、刻度范围、主次刻度、网格线等。

你要理解这些参数的作用：

**颜色 color**：区分不同组或强调重点。  
**线型 linestyle**：区分不同趋势线。  
**点型 marker**：区分不同样本点。  
**线宽 linewidth**：控制视觉强调程度。  
**标题 title**：告诉读者图在讲什么。  
**坐标轴标签 xlabel/ylabel**：告诉读者变量和单位。  
**图例 legend**：解释颜色、线型、点型代表什么。  
**刻度 ticks**：控制读数精度。  
**网格 grid**：帮助读者对齐数值。  
**坐标范围 xlim/ylim**：控制展示范围。

一张合格的图至少要有：

图标题。

x 轴和 y 轴名称。

必要时的单位。

必要时的图例。

数据来源或说明。

否则图很容易“看着漂亮但无法解释”。

## 4. 子图：用于对比多个视角

课件讲到，Matplotlib 中子图（subplots）用于在一个图表窗口中绘制多个独立图，非常适合比较多组数据或展示不同视角。`plt.subplot(nrows,ncols,index)` 中，`nrows` 和 `ncols` 表示行列数，`index` 表示第几个子图。

应用场景：

同时看四个变量的直方图。

同一变量在不同组中的分布。

模型训练集与测试集对比。

原始数据与清洗后数据对比。

课件中用 Iris 数据画了 2×2 的四个直方图，分别展示花萼长度、花萼宽度、花瓣长度、花瓣宽度的分布。

更复杂的布局可以用 `GridSpec`。课件中 `GridSpec` 允许创建不同大小和位置的子图，例如上面两个小散点图，下面一个跨两列的大散点图。

## 5. Matplotlib 的定位

Matplotlib 的优点是：

灵活。

底层控制强。

适合精细调整论文图、报告图。

几乎所有 Python 可视化库都能和它结合。

缺点是：

简单图也可能需要写较多代码。

默认美观程度一般。

统计图封装不如 Seaborn 简洁。

所以可以这样理解：

> **Matplotlib 适合“精修图”；Seaborn 适合“快速做统计探索图”。**

# 四、Seaborn：更适合统计可视化和探索性分析

课件说，Seaborn 和 Matplotlib 一样是 Python 数据可视化的重要第三方包，但它是在 Matplotlib 基础上做了更高级的 API 封装，使作图更容易、图形更美观。 Seaborn 官方介绍也说明，它是用于 Python 统计图形的库，建立在 Matplotlib 之上，并且和 Pandas 数据结构紧密集成。([seaborn.pydata.org](https://seaborn.pydata.org/tutorial/introduction.html?utm_source=chatgpt.com "An introduction to seaborn - PyData |"))

## 1. Seaborn 的核心优势

Seaborn 适合数据科学常见场景：

DataFrame 直接作图。

按类别分组自动上色。

自动处理图例。

快速画统计图。

默认风格更适合探索分析。

例如你想看 Titanic 中不同性别的生还差异，用 Seaborn 一句 `countplot(x='Survived', hue='Sex')` 就能画出按性别分组的生还/遇难数量。课件正是这样做的。

## 2. Seaborn 五种风格

课件列出 Seaborn 有五种风格：

`darkgrid`  
`dark`  
`whitegrid`  
`white`  
`ticks`

默认是 `darkgrid`。

应用判断：

**darkgrid**：适合探索分析，网格帮助读数。  
**whitegrid**：适合柱状图、箱线图、分组比较。  
**white**：适合干净报告图。  
**ticks**：适合强调坐标轴刻度。  
**dark**：适合演示或视觉背景较重的图。

## 3. Seaborn 图形分类

课件把 Seaborn 图形分为关系图、分类图、分布图、回归图、矩阵图。 官方文档也把 Seaborn 作图函数分成不同层级和类别，用于关系、分布、类别等统计可视化任务。([seaborn.pydata.org](https://seaborn.pydata.org/tutorial/function_overview.html?utm_source=chatgpt.com "Overview of seaborn plotting functions"))

常见函数可以这样记：

|目的|Seaborn 函数|用法场景|
|---|---|---|
|看散点关系|`sns.scatterplot`|两个数值变量关系|
|看趋势|`sns.lineplot`|时间序列、趋势线|
|看分布|`sns.histplot`|单变量分布|
|看密度|`sns.kdeplot`|平滑分布|
|看类别计数|`sns.countplot`|类别频数|
|看分组均值|`sns.barplot`|类别对数值的平均影响|
|看分组分布|`sns.boxplot`|类别下数值分布|
|看完整分布形状|`sns.violinplot`|组间分布比较|
|看相关矩阵|`sns.heatmap`|相关性、混淆矩阵|
|看两变量联合分布|`sns.jointplot`|散点 + 边缘分布|
|看多变量两两关系|`sns.pairplot`|探索多特征数据|

## 4. Matplotlib 和 Seaborn 怎么配合

实践中不是二选一，而是组合使用：

用 Seaborn 快速画主体图。

用 Matplotlib 调标题、坐标轴、图例、画布大小、子图布局。

比如：

`sns.boxplot(...)` 画箱线图。

`plt.title(...)` 添加标题。

`plt.xlabel(...)` 改横轴名称。

`plt.show()` 展示图。

所以你不需要背两套完全独立的体系。可以理解为：

> **Seaborn 负责快速生成统计图，Matplotlib 负责底层控制和精细调整。**

# 五、Titanic 案例：可视化如何服务数据分析

课件的 Titanic 案例非常重要，因为它展示了一个完整的探索性数据分析（exploratory data analysis, EDA）思路：**先看总体生还情况，再逐个分析性别、客舱、年龄、亲属数量、起航港等变量与生还之间的关系**。Kaggle Titanic 竞赛的任务也是根据乘客信息预测乘客是否在沉船事故中生还，目标变量是 0/1 生还标记。([Kaggle](https://www.kaggle.com/competitions/titanic?utm_source=chatgpt.com "Titanic - Machine Learning from Disaster"))

## 1. 读入数据：先确认字段含义

课件中 `train_titantic.csv` 包含 891 位乘客的基本信息和生存情况，字段包括：

`PassengerId`：乘客 ID。  
`Survived`：是否生还，0 表示遇难，1 表示生还。  
`Pclass`：客舱等级。  
`Name`：姓名。  
`Sex`：性别。  
`Age`：年龄。  
`SibSp`：兄弟姐妹/配偶数量。  
`Parch`：父母/子女数量。  
`Ticket`：船票号。  
`Fare`：票价。  
`Cabin`：客舱号。  
`Embarked`：登船港口。

这一步的重点不是代码，而是理解：

**每一行是一个乘客，每一列是乘客的一个属性，`Survived` 是目标变量。**

## 2. 先看总体生还情况

课件先计算遇难和生还人数，然后用 `countplot` 和饼图展示总体生还比例。结果显示遇难者 549 人，生还者 342 人；饼图中遇难比例约 61.62%，生还比例约 38.38%。

这个图回答的问题是：

**这次数据集中总体生还率是多少？**

但它不能回答：

谁更容易生还？

哪些因素影响生还？

要回答这些问题，需要分组可视化。

## 3. 存活情况与性别的关系

课件先画总体男女比例，再画生还乘客中的男女比例，最后用 `sns.countplot(x='Survived', hue='Sex')` 同时比较不同性别在遇难/生还中的数量。

这一组图的分析逻辑是：

第一步，看总体男女数量。

第二步，看生还者中男女数量。

第三步，看男性和女性在生还/遇难两类中的分布。

从图上可以看出，男性乘客总数更多，但生还者中女性更多。这说明性别与生还情况可能存在明显关系。

这里要注意一个分析细节：

课件图中显示的是**人数**，如果要严谨比较“女性生还率是否更高”，还应计算：

男性生还率 = 生还男性人数 / 男性总人数。

女性生还率 = 生还女性人数 / 女性总人数。

仅看人数可能会受到总体人数差异影响。

## 4. 存活情况与客舱等级的关系

课件分别画了各客舱总体人数和各客舱生还人数，并用柱状图和饼图展示不同 `Pclass` 的分布。总体上三等舱人数最多，但生还者中一等舱人数也很多。

这个分析回答：

**客舱等级是否与生还情况有关？**

但同样要注意：

不能只看“生还人数”，还要看“生还率”。

例如三等舱生还人数可能不少，是因为三等舱总人数本来就多；一等舱生还人数占一等舱总人数的比例，可能更能说明舱位等级和生还概率的关系。

所以应用中建议画：

`x='Pclass', hue='Survived'` 的 countplot。

按 `Pclass` 分组计算生还率的 barplot。

## 5. 存活情况与年龄的关系

课件用三类图分析年龄：

总体年龄直方图。

总体年龄箱线图。

按 `Survived` 分组的年龄箱线图。

堆积直方图比较生还与遇难人群的年龄分布。

这些图回答不同问题：

**直方图**：乘客年龄主要集中在哪些区间？

**箱线图**：年龄是否存在离群值？中位数是多少？

**分组箱线图**：生还者和遇难者的年龄分布是否不同？

**堆积直方图**：不同年龄段中生还/遇难数量如何分布？

分析年龄这类连续变量时，不适合只用柱状图直接数类别，因为年龄本身是连续的。更适合先看分布，再分箱或按组比较。

## 6. 存活情况与兄弟姐妹/配偶数量的关系

课件用 `sns.countplot(x='SibSp', hue='Survived')` 观察 `SibSp` 与生还之间的关系。

这个图的意义是：

看家庭同行人数是否和生还有关。

例如独自乘船、大量亲属同行、小家庭同行，可能对应不同生还情况。

应用中可以进一步构造新变量：

`FamilySize = SibSp + Parch + 1`

然后比较不同家庭规模的生还率。这是从可视化走向特征工程的典型思路。

## 7. 存活情况与起航港的关系

课件用 `sns.countplot(x='Embarked', hue='Survived')` 比较不同登船港口下的生还情况。

这个图回答：

**从不同港口登船的乘客，生还分布是否不同？**

但要谨慎解释。登船港口本身不一定直接影响生还，它可能和客舱等级、票价、乘客身份结构相关。也就是说，`Embarked` 可能是间接变量。

这体现了数据可视化中的一个重要原则：

> **图能提示关系，但不能直接证明因果。**

# 六、K 线图：金融数据可视化案例

课件最后讲金融数据可视化中的 K 线图。K 线图用于展示金融市场价格走势，常见于股票、期货、外汇等市场。每根 K 线代表一段时间内的价格变化，通常包括开盘价、收盘价、最高价和最低价。课件说明 K 线由实体、上影线和下影线构成，并可用于观察趋势、市场情绪、支撑阻力位等。

## 1. K 线图中的四个核心价格

每根 K 线通常需要四个字段：

**Open**：开盘价。  
**High**：最高价。  
**Low**：最低价。  
**Close**：收盘价。

这四个字段合称 OHLC。

课件例子中还加入了：

**Volume**：成交量。

这也是金融图中常见的辅助信息。

## 2. K 线结构怎么看

一根 K 线包括：

**实体**：开盘价与收盘价之间的区间。

**上影线**：最高价到实体上端的距离。

**下影线**：最低价到实体下端的距离。

如果收盘价高于开盘价，通常表示上涨；如果收盘价低于开盘价，通常表示下跌。不同市场会用不同颜色表示上涨/下跌，国内常见红涨绿跌，美股图表中常见绿涨红跌，因此读图时要先看图例或颜色说明。课件中也强调实体表示开盘价和收盘价之间的价格区间，上影线表示最高价与实体上端之间的价格区间，下影线表示实体下端与最低价之间的价格区间。

## 3. Python 里怎么画 K 线图

课件说 Python 中可以使用 Plotly 或 `mpf` 绘制 K 线图，并给出 `mplfinance` 的例子：创建包含 Open、High、Low、Close、Volume 的 DataFrame，再用 `mpf.plot(data,type='candle',mav=(3,6),volume=True,style='charles')` 绘制蜡烛图、均线和成交量。

`mplfinance` 官方仓库说明，它提供新的 Matplotlib 金融 API，用来更容易地创建金融图，并能很好地与 Pandas DataFrame 配合。([GitHub](https://github.com/matplotlib/mplfinance?utm_source=chatgpt.com "matplotlib/mplfinance: Financial Markets Data Visualization ..."))

## 4. K 线图的应用边界

K 线图适合：

观察价格趋势。

比较开盘价与收盘价。

观察波动范围。

配合成交量判断交易活跃度。

叠加移动平均线看趋势平滑情况。

但要注意：

K 线图不是预测保证。

单纯看形态容易主观化。

应结合交易量、基本面、风险控制、统计检验等信息。

在课程中，你主要掌握它的结构和绘制方法即可，不需要深入技术交易策略。

# 八、学习这章时最容易犯的错误

## 1. 只看数量，不看比例

Titanic 中，三等舱人数最多，所以三等舱生还人数可能也不少。但如果比较生还能力，应该看各舱位生还率。

数量回答“有多少人”。

比例回答“这一组里有多大概率”。

## 2. 用饼图展示太多类别

饼图适合 2 到 5 个类别的占比展示。类别太多时，扇区很难比较，柱状图更清楚。

Titanic 生还/遇难只有两类，用饼图可以；如果是几十个城市，不适合饼图。

## 3. 坐标轴和单位不写清楚

没有坐标轴标签的图，读者不知道变量是什么；没有单位的图，读者不知道数值含义。

例如票价是美元还是英镑？年龄是岁还是月？成交量是股数还是金额？必须标清楚。

## 4. 把相关关系说成因果关系

图可以显示“性别和生还有关”，但不能直接说“性别导致生还”。因果解释需要机制、控制变量和更严谨的分析。

## 5. 图太复杂，信息过载

一张图不要同时塞太多颜色、形状、线条和子图。可视化的目标是降低理解成本，不是展示所有信息。

## 6. 没有先处理缺失值和异常值

可视化之前应检查缺失和异常。否则图可能被错误值扭曲，例如年龄 999、票价 -1、日期错误都会导致图形误导。

# 十、最后用一句话总结

这章的核心不是“会画很多图”，而是：

> **先明确分析问题，再根据变量类型选择图表，用图发现数据规律、检查数据质量、辅助建模和表达结论。**

Matplotlib 给你底层控制能力，Seaborn 给你快速统计作图能力；Titanic 案例展示了如何用可视化做探索性分析，K 线图案例展示了特定领域中图表如何承载专业含义。
