# 第三部分第一章：无监督学习讲义

本章对应 Course 3 Week 1，主题是 **Unsupervised Learning（无监督学习）**。这一章有两条主线：

1. **Clustering（聚类）**：在没有标签的情况下，把相似的数据点自动分组。主要算法是 **K-means**。
2. **Anomaly Detection（异常检测）**：在大量正常样本中，识别“看起来不正常”的样本。主要方法是基于 **Gaussian distribution（高斯分布/正态分布）** 的密度估计。

如果把前两门课的监督学习看作“给出输入 `x` 和正确答案 `y`，让模型学会预测 `y`”，那么这一章的无监督学习就是“只给出输入 `x`，让算法自己发现数据中的结构”。这也是本章最重要的转变：我们不再告诉算法什么是正确答案，而是设计一个合理的数学目标，让算法从数据自身的形状、密度或相似性中找到规律。

---

## 1. 从监督学习到无监督学习

在监督学习（supervised learning）中，训练集通常写成：

$$
\{(x^{(1)}, y^{(1)}), (x^{(2)}, y^{(2)}), \ldots, (x^{(m)}, y^{(m)})\}
$$

也就是说，每个样本既有输入特征 `x`，也有标签 `y`。例如在二分类问题中，`y=0` 和 `y=1` 可以表示两个类别。算法的目标是学习一个函数，让它能根据新的 `x` 预测对应的 `y`。

而无监督学习（unsupervised learning）的训练集通常只有：

$$
\{x^{(1)}, x^{(2)}, \ldots, x^{(m)}\}
$$

这里没有 `y`。因此我们无法说某个点的“正确类别”是什么，也无法像逻辑回归或神经网络那样直接拟合一个标签。算法要做的是寻找数据中“有意思的结构”（interesting structure）。本章的两个结构分别是：

- 数据点是否自然形成若干相似群体，这就是聚类。
- 某个点是否落在正常数据很少出现的区域，这就是异常检测。

这两个任务看似都没有标签，但它们的思路不同。聚类关心“哪些点彼此相似”；异常检测关心“哪些点不像正常样本”。

---

## 2. 聚类：让数据自己分组

### 2.1 什么是聚类

**Clustering（聚类）** 的目标是自动寻找一组数据点中的相似点，并把它们分成若干组。每一组叫一个 **cluster（簇）**。

例如，给定一个只有 `x_1` 和 `x_2` 两个特征的数据集，如果数据点在平面上明显分成两团，那么聚类算法可能会把它们分成两个簇。这里没有真实标签告诉算法“这个点属于第一类，那个点属于第二类”，算法只能根据点之间的距离和分布来推断。

课程中提到的聚类应用包括：

- 新闻聚合：把相似主题的新闻文章放在一起。
- 市场细分（market segmentation）：把有相似需求或行为的用户分成不同群体。
- DNA 或基因表达数据分析：寻找具有相似生物特征的样本。
- 天文学数据分析：把空间中可能属于同一结构的天体聚在一起。

聚类不是“预测标签”，而是“发现结构”。因此，有时同一个数据集可以有多种合理分法。

---

## 3. K-means 的直观理解

本章最重要的聚类算法是 **K-means clustering（K 均值聚类）**。这里的 `K` 表示你希望算法找出多少个簇。

K-means 的核心思想非常简单：每个簇用一个中心点表示，这个中心点叫 **centroid（簇中心）**。算法反复做两件事：

1. **Assign points to centroids（把点分配给最近的簇中心）**
2. **Move centroids（把簇中心移动到所属点的平均位置）**

假设我们希望把数据分成两个簇，即 `K=2`。算法可以这样开始：

1. 随机选择两个初始簇中心，记为 `\mu_1` 和 `\mu_2`。
2. 对每个训练样本 `x^{(i)}`，计算它离哪个簇中心最近，并把它分给那个簇。
3. 对每个簇，把簇中心移动到该簇所有点的平均位置。
4. 重复第 2 步和第 3 步，直到点的分配和簇中心的位置不再明显变化。

这个过程的直觉是：簇中心一开始可能猜得不好，但只要不断“先分配，再求平均”，簇中心通常会逐渐移动到每一团数据的中心位置。

当算法继续迭代但结果不再改变时，我们说 K-means **converged（收敛）**。

---

## 4. K-means 算法的数学形式

设训练集为：

$$
x^{(1)}, x^{(2)}, \ldots, x^{(m)}
$$

每个样本 `x^{(i)}` 是一个 `n` 维向量。若希望分成 `K` 个簇，则簇中心为：

$$
\mu_1, \mu_2, \ldots, \mu_K
$$

每个 `\mu_k` 也是一个 `n` 维向量，和样本特征维度相同。

### 4.1 第一步：簇分配

对每个样本 `x^{(i)}`，找到距离它最近的簇中心：

$$
c^{(i)} = \arg\min_{k \in \{1,\ldots,K\}} \|x^{(i)} - \mu_k\|^2
$$

这里：

- `c^{(i)}` 表示第 `i` 个样本被分配到哪个簇。
- `\mu_k` 表示第 `k` 个簇中心。
- `\|x^{(i)} - \mu_k\|^2` 是样本到簇中心的平方欧氏距离。

为什么用平方距离？因为最小距离和最小平方距离对应的是同一个簇中心，但平方距离更方便写成优化目标。

### 4.2 第二步：移动簇中心

当所有点都已经分配给某个簇后，对每个簇 `k`，重新计算簇中心：

$$
\mu_k = \frac{1}{|C_k|}\sum_{i \in C_k}x^{(i)}
$$

这里：

- `C_k` 是所有被分配到第 `k` 个簇的样本集合。
- `|C_k|` 是该簇中的样本数量。

也就是说，新的簇中心就是这个簇中所有点的平均值。

### 4.3 空簇问题

有时某个簇中心附近没有任何点，导致 `C_k` 为空。这时无法计算平均值。常见处理方式有两种：

- 直接删除这个簇，最终得到 `K-1` 个簇。
- 如果必须保留 `K` 个簇，就随机重新初始化这个簇中心。

课程中提到，更常见的做法是删除没有被分配样本的簇。

## 5. K-means 在优化什么

K-means 看起来像是在做一个启发式过程，但它其实在优化一个明确的目标函数。这个目标函数叫 **distortion function（失真函数）** 或 **K-means cost function（K-means 代价函数）**：

$$
J(c^{(1)},\ldots,c^{(m)},\mu_1,\ldots,\mu_K)
=
\frac{1}{m}\sum_{i=1}^{m}\|x^{(i)}-\mu_{c^{(i)}}\|^2
$$

它表示：每个样本到自己所属簇中心的平方距离的平均值。

这个公式非常重要，因为它解释了 K-means 为什么会收敛：

- 在簇分配步骤中，簇中心 `\mu_1,\ldots,\mu_K` 固定，算法为每个点选择最近的簇中心，因此会让 `J` 不增大。
- 在移动簇中心步骤中，分配结果 $c^{(1)},\ldots,c^{(m)}$固定，算法把中心移动到所属点的平均值，这也会让 `J` 不增大。

因此，每一次迭代后，失真函数 `J` 都应该下降或保持不变。如果你在代码里发现 `J` 上升，通常意味着实现有 bug。

需要注意的是：K-means 收敛不代表找到全局最优解。它可能收敛到一个 **local optimum（局部最优）**。

## 6. K-means 的初始化

K-means 的结果很依赖初始簇中心。不同的初始点可能导致不同的最终聚类结果。

常见初始化方法是：

1. 随机打乱训练样本的索引。
2. 从训练集中随机选出 `K` 个样本。
3. 把这 `K` 个样本作为初始簇中心。

要求通常是 `K <= m`，因为如果簇的数量比训练样本还多，就不可能让每个簇至少有一个样本。

为了减少局部最优的影响，实际使用 K-means 时通常会：

1. 用不同的随机初始化运行 K-means 多次。
2. 每次运行到收敛后，计算失真函数 `J`。
3. 选择 `J` 最小的那一次结果。

课程中提到，运行 50 到 1000 次随机初始化都很常见。次数越多，越可能找到更好的局部最优，但计算成本也会增加。

## 7. 如何选择 K

K-means 需要你提前指定 `K`，也就是簇的数量。但这个问题经常没有唯一正确答案。

### 7.1 Elbow method（肘部法则）

一种常见方法是对多个 `K` 运行 K-means，然后画出：

$$
K \quad \text{vs.} \quad J
$$

通常 `K` 越大，失真函数 `J` 越小，因为更多簇可以更细地贴合数据。如果曲线上出现明显的“肘部”，例如从 `K=1` 到 `K=3` 下降很快，之后下降变慢，那么可以考虑选择肘部对应的 `K`。

但课程也强调：肘部法则经常不可靠。很多时候曲线只是平滑下降，没有明显拐点。

### 7.2 根据下游目标选择 K

更实用的方法是根据后续用途选择 `K`。

例如用 K-means 决定 T 恤尺码：

- `K=3` 可以对应 S/M/L。
- `K=5` 可以对应 XS/S/M/L/XL。

哪种更好，不只取决于数据本身，还取决于商业目标：更多尺码可能更贴身，但制造、库存和运输成本更高。

在图像压缩中也是类似：

- `K` 越大，保留的颜色越多，图像质量越好。
- `K` 越小，压缩率越高，但图像细节损失更多。

因此，选择 `K` 本质上是在模型质量、成本和实际需求之间做权衡。

## 8. K-means 实验代码讲解

本章的 K-means 练习文件是：

```text
k-means/C3_W1_KMeans_Assignment.ipynb
```

这个实验分两部分：

1. 在二维玩具数据上实现 K-means。
2. 用 K-means 做图像压缩。

### 8.1 `find_closest_centroids`

这个函数对应 K-means 的第一步：给每个样本找最近的簇中心。

输入：

- `X`: 形状为 `(m, n)`，表示 `m` 个样本、每个样本 `n` 个特征。
- `centroids`: 形状为 `(K, n)`，表示 `K` 个簇中心。

输出：

- `idx`: 形状为 `(m,)`，第 `i` 个元素表示第 `i` 个样本被分配到哪个簇。

课程公式使用 `1` 到 `K` 表示簇编号，但 Python 从 `0` 开始编号，所以实验中簇编号是 `0` 到 `K-1`。

实现逻辑是：

```python
for i in range(X.shape[0]):
    distance = []
    for j in range(centroids.shape[0]):
        norm_ij = np.linalg.norm(X[i] - centroids[j])
        distance.append(norm_ij)
    idx[i] = np.argmin(distance)
```

这里 `np.linalg.norm(X[i] - centroids[j])` 计算欧氏距离，`np.argmin(distance)` 找到最小距离对应的簇中心索引。

虽然数学公式中常写平方距离，但在找最近簇中心时，用距离或平方距离得到的索引相同。

### 8.2 `compute_centroids`

这个函数对应 K-means 的第二步：根据分配结果重新计算簇中心。

输入：

- `X`: 数据矩阵。
- `idx`: 每个样本所属簇的编号。
- `K`: 簇数量。

输出：

- `centroids`: 新的簇中心矩阵，形状为 `(K, n)`。

核心代码是：

```python
for k in range(K):
    points = X[idx == k]
    centroids[k] = np.mean(points, axis=0)
```

这里 `X[idx == k]` 是布尔索引，表示取出所有被分到第 `k` 个簇的样本。`np.mean(points, axis=0)` 表示对这些样本按列求平均，也就是对每个特征分别求平均。

### 8.3 `run_kMeans`

实验中的 `run_kMeans` 把前两个函数放进循环：

```python
idx = find_closest_centroids(X, centroids)
centroids = compute_centroids(X, idx, K)
```

重复多次后，簇中心逐渐移动到稳定位置。实验还会画出簇中心移动轨迹，帮助你看到 K-means 是如何一步步收敛的。

### 8.4 随机初始化函数

`kMeans_init_centroids(X, K)` 的逻辑是：

```python
randidx = np.random.permutation(X.shape[0])
centroids = X[randidx[:K]]
```

它先随机打乱样本索引，然后选前 `K` 个样本作为初始簇中心。这样可以避免重复选择同一个样本。

### 8.5 图像压缩：K-means 的一个直观应用

实验中读取 `bird_small.png`，原图是一个 `128 x 128 x 3` 的 RGB 图片。每个像素有三个通道：

- R: red
- G: green
- B: blue

因此，每个像素可以看作一个三维向量：

$$
[R, G, B]
$$

图像压缩的思路是：

1. 把图片 reshape 成 `(16384, 3)` 的矩阵，因为 `128 x 128 = 16384`。
2. 把每个像素当作一个三维数据点。
3. 用 K-means 找出 `K=16` 个代表性颜色。
4. 把每个像素替换为离它最近的代表性颜色。

原始 RGB 图像每个像素需要 24 bits，因为 3 个通道各 8 bits。原始大小为：

$$
128 \times 128 \times 24 = 393216 \text{ bits}
$$

压缩后需要存储：

- 16 个颜色，每个颜色 24 bits：`16 x 24`
- 每个像素只需存储 16 个颜色中的索引，`16=2^4`，所以每个像素 4 bits：`128 x 128 x 4`

总大小为：

$$
16 \times 24 + 128 \times 128 \times 4 = 65920 \text{ bits}
$$

大约压缩了 6 倍。

这个例子说明，K-means 不一定只用于“发现自然类别”。它也可以用于 **vector quantization（向量量化）**：用少数代表点近似大量原始点。

## 9. 异常检测：找出不寻常的样本

本章第二个主题是 **Anomaly Detection（异常检测）**。

异常检测的目标是：从大量正常样本中学习“正常数据大概长什么样”，然后判断一个新样本是否过于罕见。

课程中的典型例子是飞机发动机检测。假设每个发动机有两个特征：

- `x_1`: 发热量
- `x_2`: 振动强度

大多数发动机是正常的，坏发动机很少。因此我们很难收集大量“坏发动机”的标签数据。异常检测的思路是：先根据大量正常发动机建立一个概率模型 `p(x)`，然后判断新发动机的特征是否落在低概率区域。

如果：

$$
p(x_{\text{test}}) < \epsilon
$$

就认为它是异常样本；如果：

$$
p(x_{\text{test}}) \ge \epsilon
$$

就认为它看起来正常。

这里 `\epsilon` 是一个小阈值，读作 epsilon。

### 9.1 异常检测的应用

课程中提到的应用包括：

- 欺诈检测（fraud detection）：识别异常账户行为、异常交易模式。
- 制造业质检：识别可能有缺陷的发动机、电路板、手机或其他产品。
- 数据中心监控：发现行为异常的机器，例如硬件故障或被攻击。
- 通信网络监控：发现行为异常的基站或设备。

这些场景的共同点是：正常样本很多，异常样本很少，而且未来出现的异常可能和过去见过的异常不同。

## 10. 高斯分布：异常检测的概率基础

为了做异常检测，课程使用 **Gaussian distribution（高斯分布）**，也叫 **normal distribution（正态分布）**。

如果一个随机变量 `x` 服从均值为 `\mu`、方差为 `\sigma^2` 的高斯分布，记作：

$$
x \sim \mathcal{N}(\mu, \sigma^2)
$$

它的概率密度函数为：

$$
p(x;\mu,\sigma^2)
=
\frac{1}{\sqrt{2\pi}\sigma}
\exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)
$$

其中：

- `\mu` 是 mean（均值），控制分布中心。
- `\sigma` 是 standard deviation（标准差），控制曲线宽度。
- `\sigma^2` 是 variance（方差）。

直觉上：

- `\mu` 变大，整条曲线向右移动。
- `\sigma` 变小，曲线更窄、更高。
- `\sigma` 变大，曲线更宽、更矮。

概率密度曲线下面积总是 1。因此曲线变窄时会变高，曲线变宽时会变低。

### 10.1 从数据估计高斯分布参数

如果我们有 `m` 个一维样本：

$$
x^{(1)}, x^{(2)}, \ldots, x^{(m)}
$$

均值估计为：

$$
\mu = \frac{1}{m}\sum_{i=1}^{m}x^{(i)}
$$

方差估计为：

$$
\sigma^2 = \frac{1}{m}\sum_{i=1}^{m}(x^{(i)}-\mu)^2
$$

有些统计课会用 `1/(m-1)` 估计方差，但课程这里使用 `1/m`。在机器学习异常检测中，这样做通常没有问题。

## 11. 多特征异常检测算法

实际问题通常不止一个特征。假设每个样本有 `n` 个特征：

$$
x = [x_1, x_2, \ldots, x_n]
$$

课程采用的模型是：

$$
p(x)=p(x_1;\mu_1,\sigma_1^2)\cdot p(x_2;\mu_2,\sigma_2^2)\cdots p(x_n;\mu_n,\sigma_n^2)
$$

也可以写成：

$$
p(x)=\prod_{j=1}^{n}p(x_j;\mu_j,\sigma_j^2)
$$

这个写法隐含了一个假设：各个特征之间近似独立（independence assumption）。课程也强调，即使这个假设不完全成立，算法在很多实际问题中仍然有效。

### 11.1 训练阶段

给定训练集：

$$
x^{(1)}, x^{(2)}, \ldots, x^{(m)}
$$

对每个特征 `j`，估计：

$$
\mu_j = \frac{1}{m}\sum_{i=1}^{m}x_j^{(i)}
$$

$$
\sigma_j^2 =
\frac{1}{m}\sum_{i=1}^{m}(x_j^{(i)}-\mu_j)^2
$$

训练结束后，我们就得到了每个特征的高斯分布参数。

### 11.2 预测阶段

对一个新样本 `x`：

1. 计算每个特征在对应高斯分布下的概率密度。
2. 把这些概率密度相乘，得到 `p(x)`。
3. 和阈值 `\epsilon` 比较。

判定规则是：

$$
\begin{cases}
p(x)<\epsilon, & \text{anomaly}\\
p(x)\ge\epsilon, & \text{normal}
\end{cases}
$$

这个算法的直觉是：如果一个样本有一个或多个特征非常大或非常小，那么对应的概率密度会很低。多个特征概率相乘后，整体 `p(x)` 会很小，于是算法把它标记为异常。

## 12. 如何开发和评估异常检测系统

异常检测虽然主要从无标签数据中学习，但在实际开发中，如果能拿到少量已知异常样本，就应该用它们来构建交叉验证集和测试集。这样可以对算法做“实数评估”（real number evaluation），也就是用一个明确指标判断系统有没有变好。

### 12.1 数据集划分

假设有：

- 10000 个正常发动机样本。
- 20 个已知异常发动机样本。

一种划分方式是：

- 训练集：6000 个正常样本。
- 交叉验证集：2000 个正常样本 + 10 个异常样本。
- 测试集：2000 个正常样本 + 10 个异常样本。

训练集主要用于估计 `p(x)`，通常假设训练集都是正常样本。即使少量异常样本混进训练集，算法一般也还能工作。

交叉验证集用于：

- 选择阈值 `\epsilon`。
- 调整特征。
- 比较不同模型设定。

测试集用于最终评估模型在未来数据上的表现。

如果异常样本极少，例如只有 2 个，那么可以只设置训练集和交叉验证集，把所有异常样本放进交叉验证集。这种做法的缺点是没有独立测试集，容易把 `\epsilon` 和特征选择过拟合到交叉验证集。

### 12.2 为什么不能只看准确率

异常检测通常是高度偏斜数据（skewed data）。例如交叉验证集中有 2000 个正常样本和 10 个异常样本。如果模型永远预测“正常”，准确率也会很高，但它完全没有发现异常。

因此更适合使用：

- **Precision（精确率）**
- **Recall（召回率）**
- **F1 score（F1 分数）**

定义如下：

$$
\text{precision}=\frac{tp}{tp+fp}
$$

$$
\text{recall}=\frac{tp}{tp+fn}
$$

$$
F_1=\frac{2\cdot \text{precision}\cdot \text{recall}}{\text{precision}+\text{recall}}
$$

其中：

- `tp`（true positive）：真实异常，且被正确预测为异常。
- `fp`（false positive）：真实正常，但被错误预测为异常。
- `fn`（false negative）：真实异常，但没有被检测出来。

在工程上，调 `\epsilon` 的本质就是调 precision 和 recall 的平衡：

- `\epsilon` 越大，越容易报异常，recall 可能上升，但 false positive 也可能变多。
- `\epsilon` 越小，越谨慎，false positive 可能减少，但可能漏掉异常。

## 13. 异常检测 vs 监督学习

一个容易困惑的问题是：如果我们已经有少量异常样本，为什么不用监督学习？

课程给出的判断标准很实用。

### 13.1 更适合异常检测的情况

使用异常检测通常更合适，当：

- 正样本，也就是异常样本非常少，例如 0 到 20 个。
- 负样本，也就是正常样本很多。
- 异常类型很多，而且未来异常可能和过去见过的异常完全不同。
- 你更容易建模“正常是什么样”，而不是建模“异常是什么样”。

典型例子：

- 新型金融欺诈。
- 新型制造缺陷。
- 服务器被入侵后的未知异常行为。
- 飞机发动机出现过去没见过的故障模式。

异常检测的思路是：只要它不像正常样本，就值得被标记出来。

### 13.2 更适合监督学习的情况

使用监督学习通常更合适，当：

- 正样本和负样本都足够多。
- 未来的正样本和训练集中见过的正样本相似。
- 类别本身比较稳定。

典型例子：

- 垃圾邮件分类（spam classification）：未来垃圾邮件通常和过去垃圾邮件有相似模式。
- 天气预测：输出类别相对固定。
- 根据症状判断某种已知疾病：如果有足够正负样本，监督学习更自然。
- 制造业中检测已知缺陷，例如手机外壳划痕。

简化判断方式是：

- 如果你有足够多的异常样本，并且未来异常会像过去异常，考虑监督学习。
- 如果你异常样本很少，并且未来异常可能是全新的，考虑异常检测。

## 14. 异常检测中的特征选择

课程特别强调：异常检测里特征选择比很多监督学习任务更关键。

原因是，监督学习有标签 `y` 作为信号。即使你给了几个不太有用的特征，模型可能也能通过标签学会忽略它们。但异常检测主要从无标签正常数据中估计概率分布，算法很难自动知道哪些特征应该忽略。

### 14.1 让特征更接近高斯分布

由于算法用高斯分布拟合每个特征，如果某个特征分布非常偏斜，可以尝试做变换，让它更接近钟形曲线。

常见变换包括：

$$
x \leftarrow \log(x)
$$

$$
x \leftarrow \log(x+c)
$$

$$
x \leftarrow \sqrt{x}
$$

$$
x \leftarrow x^{0.4}
$$

如果 `x` 可能为 0，则不能直接取 `log(x)`，可以改用 `log(x + 0.001)` 或 `log(x+c)`。

实践中常用方法是：

1. 对特征画 histogram（直方图）。
2. 尝试不同变换。
3. 看变换后的分布是否更接近对称的高斯形状。

注意：对训练集用了什么变换，对交叉验证集和测试集也必须使用同样的变换。

### 14.2 用错误分析创造新特征

训练模型后，最常见的问题是：某些异常样本的 `p(x)` 仍然很高，导致模型没有把它们标为异常。

这时应该做 error analysis（错误分析）：

1. 找出交叉验证集中漏检的异常样本。
2. 观察这些样本为什么在业务上显得异常。
3. 设计新特征，让这些异常在新特征上表现得更极端。

例如欺诈检测中，如果某个用户的交易次数并不异常，但打字速度极快，那么可以加入：

$$
x_{\text{typing speed}}
$$

例如数据中心监控中，一台机器 CPU load 很高，但 network traffic 很低。如果单看 CPU 或网络流量都不极端，可以构造比值特征：

$$
x_5 = \frac{\text{CPU load}}{\text{network traffic}}
$$

甚至可以尝试：

$$
x_6 = \frac{(\text{CPU load})^2}{\text{network traffic}}
$$

新特征的目标不是“越复杂越好”，而是让正常样本的 `p(x)` 仍然较高，同时让异常样本的 `p(x)` 变低。

## 15. 异常检测实验代码讲解

本章的异常检测练习文件是：

```text
Anomaly Detection/C3_W1_Anomaly_Detection.ipynb
```

实验任务是检测服务器异常行为。二维数据集中的两个特征是：

- throughput（吞吐量，mb/s）
- latency（响应延迟，ms）

训练集中有 `m=307` 个服务器样本，大多数是正常样本。实验先在二维数据上可视化算法，再扩展到 11 维高维数据集。

### 15.1 `estimate_gaussian`

这个函数用于估计每个特征的均值和方差。

输入：

- `X`: 形状为 `(m, n)` 的数据矩阵。

输出：

- `mu`: 形状为 `(n,)`，每个特征的均值。
- `var`: 形状为 `(n,)`，每个特征的方差。

实现是：

```python
mu = 1 / m * np.sum(X, axis=0)
var = 1 / m * np.sum((X - mu) ** 2, axis=0)
```

这里 `axis=0` 表示沿着样本维度求和，因此得到的是每一列，也就是每个特征的统计量。

也可以写成：

```python
mu = np.mean(X, axis=0)
var = np.var(X, axis=0)
```

但实验中显式写公式，更能看清数学含义。

### 15.2 `multivariate_gaussian`

工具函数 `multivariate_gaussian(X, mu, var)` 会计算每个样本的概率密度 `p(x)`。当 `var` 是一个向量时，它会被转换成对角协方差矩阵，这和课程中“每个特征分别用一个高斯分布，再把概率相乘”的做法一致。

直观理解即可：它根据 `mu` 和 `var` 判断每个样本落在当前高斯模型中的概率有多高。

### 15.3 `select_threshold`

这个函数用于在交叉验证集上选择最佳阈值 `epsilon`。

输入：

- `y_val`: 交叉验证集真实标签，异常为 1，正常为 0。
- `p_val`: 交叉验证集中每个样本的概率密度。

核心逻辑是：

```python
predictions = (p_val < epsilon)

tp = np.sum((predictions == 1) & (y_val == 1))
fp = np.sum((predictions == 1) & (y_val == 0))
fn = np.sum((predictions == 0) & (y_val == 1))

prec = tp / (tp + fp)
rec = tp / (tp + fn)
F1 = 2 * prec * rec / (prec + rec)
```

实验会在 `min(p_val)` 到 `max(p_val)` 之间尝试很多 `epsilon`，选择 F1 分数最高的阈值。

二维数据集的预期结果大约是：

```text
Best epsilon found using cross-validation: 8.99e-05
Best F1 on Cross Validation Set: 0.875
```

高维 11 特征数据集的预期结果大约是：

```text
Best epsilon found using cross-validation: 1.38e-18
Best F1 on Cross Validation Set: 0.615385
# Anomalies found: 117
```

这些结果说明：异常检测在高维空间中仍可使用，但阈值的数量级可能非常小，因为多个特征的概率密度相乘后会迅速变成极小值。

## 17. 本章易错点

1. **K-means 是无监督学习，不是分类。**  
   聚类结果中的“簇编号”不是标签。簇 0、簇 1、簇 2 只是算法给出的分组索引，没有天然语义。

2. **K-means 需要提前指定 K。**  
   算法不会自动告诉你唯一正确的簇数量。很多时候应根据下游目标选择 K。

3. **K-means 会收敛，但可能收敛到局部最优。**  
   因此实际使用时应多次随机初始化，并选择失真函数最低的结果。

4. **不要用“让 J 最小”来选择 K。**  
   因为 K 越大，J 通常越小。若只最小化 J，往往会选择过大的 K。

5. **异常检测不是简单的二分类。**  
   它主要学习正常样本的概率模型，适合异常样本极少且未来异常可能未知的场景。

6. **异常检测中准确率常常没有意义。**  
   因为异常样本太少，永远预测正常也可能得到很高准确率。应关注 precision、recall 和 F1。

7. **特征变换必须一致。**  
   如果训练集用了 `log(x+c)`，交叉验证集、测试集和未来线上数据也必须用同样的变换。

8. **概率很小不一定是 bug。**  
   在高维异常检测中，多个概率密度相乘后 `p(x)` 可能非常小，所以 `epsilon` 也可能是 `1e-18` 这样的数量级。

## 18. 一句话总结

本章的核心是：当没有标签 `y` 时，机器学习仍然可以通过“相似性”和“概率密度”发现数据结构。K-means 用距离把数据分成若干簇，异常检测用高斯概率模型找出低概率样本。前者适合理解数据内部的群体结构，后者适合在大量正常样本中发现罕见、不寻常、值得进一步检查的事件。
