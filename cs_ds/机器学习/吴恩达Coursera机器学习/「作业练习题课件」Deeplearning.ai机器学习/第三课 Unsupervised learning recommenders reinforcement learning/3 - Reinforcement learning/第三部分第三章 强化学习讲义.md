# 第三部分第三章：强化学习讲义

本章对应 Course 3 Week 3，主题是 **Reinforcement Learning（强化学习）**。这是整个机器学习专项课程的最后一章，也是和前面监督学习、无监督学习、推荐系统最不同的一类问题。

> 给定输入 `x` 和正确答案 `y`，学习从 `x` 到 `y` 的映射。

强化学习不是这样。强化学习通常没有人告诉算法“每一步正确动作是什么”。我们只设计一个 **reward function（奖励函数）**，告诉智能体做得好还是不好，然后让它自己通过尝试学习一套决策策略。

这一章的主线是：

1. 什么是强化学习：state、action、reward、return、policy。

3. `Q(s,a)` 和 Bellman Equation：强化学习算法的核心。
4. 从离散状态到连续状态：Lunar Lander 月球着陆器。
5. Deep Q-Learning / DQN：用神经网络近似 `Q(s,a)`。
6. DQN 的关键改进：target network、experience replay、epsilon-greedy、mini-batch、soft update。
7. 强化学习的现实位置：很重要，但比监督学习更难落地、更依赖仿真和工程调参。

## 1. 强化学习和监督学习有什么不同

课程一开始用自动直升机作为例子。直升机有位置、方向、速度、角速度等信息，程序要根据这些信息决定如何推动控制杆，使它稳定飞行，甚至完成倒飞等动作。

如果用监督学习，我们需要构造很多训练样本：

$$
(x, y)
$$

其中：

- `x` 是直升机当前状态，例如位置、速度、姿态。
- `y` 是专家飞行员在该状态下应该采取的最佳动作。

问题是：在很多控制任务中，所谓“最佳动作”并不唯一，也很难由人类逐帧标注。某一刻应该稍微左倾还是多左倾？应该加一点推力还是加很多推力？这些标签很难精确定义。

强化学习换了一个思路：不告诉算法每一步应该做什么，而是告诉它结果好不好。

例如：

- 飞得稳定，每秒给 `+1` 奖励。
- 飞得不好，给负奖励。
- 坠毁，给 `-1000` 这样的大惩罚。

这就像训练小狗：你不需要精确告诉它每一步该怎么迈腿，而是在它做对时奖励，做错时惩罚。强化学习算法的任务是：通过不断尝试，学会多做能带来高奖励的事，少做会带来低奖励或惩罚的事。

## 2. 强化学习的基本框架

强化学习通常描述为 **agent-environment loop（智能体-环境循环）**。

在每个时间步：

1. 智能体观察当前状态 `S_t`。
2. 智能体根据策略选择动作 `A_t`。
3. 环境根据动作发生变化。
4. 智能体收到奖励 `R_t`。
5. 环境进入新状态 `S_{t+1}`。

可以写成一个经验元组：

$$
(S_t, A_t, R_t, S_{t+1})
$$

在 Lunar Lander 作业里还会额外保存 `done`：

$$
(S_t, A_t, R_t, S_{t+1}, done)
$$

其中 `done=True` 表示 episode（回合）结束。

## 3. Mars Rover 例子：最小的强化学习问题

为了避免一开始就处理复杂机器人，课程用 Mars Rover 火星车建立直觉。

假设火星车只能处在 6 个离散位置之一：

```text
state 1, state 2, state 3, state 4, state 5, state 6
```

火星车从 `state 4` 开始。每一步可以选择两个动作：

- `left`
- `right`

奖励函数设定为：

- `state 1` 有重要科学价值，奖励 `100`。
- `state 6` 也有科学价值，但较低，奖励 `40`。
- 中间状态 `state 2` 到 `state 5` 奖励为 `0`。

`state 1` 和 `state 6` 是 **terminal state（终止状态）**。到达终止状态后，这一天就结束，不再继续获得奖励。

例如从 `state 4` 一直向左：

```text
state 4 -> state 3 -> state 2 -> state 1
```

奖励序列是：

```text
0, 0, 0, 100
```

如果从 `state 4` 一直向右：

```text
state 4 -> state 5 -> state 6
```

奖励序列是：

```text
0, 0, 40
```

注意，强化学习关心的不是某一步动作本身，而是一串动作最终带来的总效果。

## 4. Return：折扣后的累计奖励

强化学习的目标不是最大化某一步奖励，而是最大化长期收益。这个长期收益叫 **return（回报）**。

如果奖励序列为：

$$
R_1, R_2, R_3, R_4, \ldots
$$

回报定义为：

$$
G = R_1 + \gamma R_2 + \gamma^2 R_3 + \gamma^3 R_4 + \cdots
$$

这里 `\gamma` 是 **discount factor（折扣因子）**，通常是小于 1 的正数，例如：

- `0.9`
- `0.99`
- `0.999`

课程在 Mars Rover 例子中为了便于计算，常用：

$$
\gamma = 0.5
$$

折扣因子的含义是：越晚得到的奖励，权重越小。它让智能体有一点“急性子”，更偏好早点拿到奖励。

### 4.1 例子：从 state 4 往左

从 `state 4` 一直往左：

```text
0, 0, 0, 100
```

当 `\gamma=0.5` 时：

$$
G = 0 + 0.5 \times 0 + 0.5^2 \times 0 + 0.5^3 \times 100 = 12.5
$$

当 `\gamma=0.9` 时：

$$
G = 0 + 0.9 \times 0 + 0.9^2 \times 0 + 0.9^3 \times 100 = 72.9
$$

折扣因子越接近 1，智能体越愿意等待远处的大回报；折扣因子越小，智能体越偏好近处的奖励。

### 4.2 负奖励下的直觉

如果奖励是负数，折扣因子会让智能体倾向于把负奖励推迟到更远的未来。

例如今天支付 10 美元比几年后支付 10 美元更“痛”，因为未来的钱经过折扣后当前价值更低。这也是为什么折扣因子在金融问题里有自然解释。

## 5. Policy：策略就是状态到动作的函数

强化学习的目标是学习一个 **policy（策略）**，通常记为：

$$
\pi(s)
$$

策略输入一个状态 `s`，输出应该采取的动作 `a`。

例如在 Mars Rover 中，一个策略可以是：

```text
state 2 -> left
state 3 -> left
state 4 -> left
state 5 -> right
```

这个策略的含义是：

- 离左侧高奖励比较近时往左。
- 在 `state 5` 时，因为右侧奖励 `40` 很近，所以往右。

强化学习的目标可以概括为：

> 找到一个策略 `\pi(s)`，使得智能体从任意状态出发时的 return 尽可能大。

## 6. MDP：Markov Decision Process

课程把强化学习问题形式化为 **Markov Decision Process（马尔可夫决策过程，MDP）**。

MDP 包含：

- states（状态）
- actions（动作）
- rewards（奖励）
- discount factor（折扣因子）
- transition（状态转移）
- policy（策略）

“Markov” 的含义是：

> 未来只取决于当前状态，而不取决于你是如何到达当前状态的。

也就是说，如果当前状态 `s` 已经完整描述了环境，那么历史轨迹不需要再额外考虑。

在直升机例子中：

- state：位置、姿态、速度、角速度等。
- action：控制杆或推力控制。
- reward：飞得稳定给正奖励，坠毁给大负奖励。

在国际象棋中：

- state：棋盘局面。
- action：合法走法。
- reward：赢棋 `+1`，输棋 `-1`，平局 `0`。

MDP 是强化学习中描述问题的标准语言。

## 7. State-Action Value Function：状态-动作价值函数

强化学习最重要的量之一是 **state-action value function（状态-动作价值函数）**，记为：

$$
Q(s,a)
$$

它的定义是：

> 如果从状态 `s` 开始，先采取动作 `a` 一次，然后之后都采取最优行动，那么最终能得到的 return 是多少。

这个函数也叫：

- **Q function（Q 函数）**
- **optimal Q function（最优 Q 函数）**
- 有时文献中写作 `Q^*(s,a)`

定义里有一点看起来像循环：如果我们还不知道最优策略，怎么知道“之后都最优行动”的 return？课程后面用 Bellman 方程和 DQN 来解决这个问题。

---

## 8. Q 函数例子

继续 Mars Rover，取：

$$
\gamma=0.5
$$

假设最优策略是：

```text
state 2 -> left
state 3 -> left
state 4 -> left
state 5 -> right
```

### 8.1 `Q(state 2, right)`

如果在 `state 2` 先向右：

```text
state 2 -> state 3 -> state 2 -> state 1
```

奖励序列是：

```text
0, 0, 0, 100
```

所以：

$$
Q(2, right)=0+0.5\times0+0.5^2\times0+0.5^3\times100=12.5
$$

这并不表示 `right` 是好动作，只是忠实记录“如果先这么做，再最优行动”会得到多少 return。

### 8.2 `Q(state 2, left)`

如果在 `state 2` 先向左：

```text
state 2 -> state 1
```

奖励序列是：

```text
0, 100
```

所以：

$$
Q(2,left)=0+0.5\times100=50
$$

因此在 `state 2`，`left` 比 `right` 更好。

### 8.3 用 Q 函数选动作

如果我们已经知道所有动作的 Q 值，那么最优策略非常简单：

$$
\pi(s)=\arg\max_a Q(s,a)
$$

也就是说，在当前状态 `s` 下，计算每个动作的 `Q(s,a)`，选择 Q 值最大的动作。

最优可获得回报是：

$$
\max_a Q(s,a)
$$

这就是为什么强化学习算法经常把核心目标设为：学习一个好的 Q 函数。

## 10. Bellman Equation：强化学习的核心方程

如果 Q 函数这么重要，那么如何计算它？

核心工具是 **Bellman Equation（贝尔曼方程）**：

$$
Q(s,a)=R(s)+\gamma\max_{a'}Q(s',a')
$$

其中：

- `s` 是当前状态。
- `a` 是当前采取的动作。
- `R(s)` 是当前状态的即时奖励。
- `s'` 是采取动作 `a` 后到达的新状态。
- `a'` 是在新状态 `s'` 下可采取的下一步动作。
- `\max_{a'}Q(s',a')` 表示从新状态出发能获得的最佳未来回报。

Bellman 方程的直觉是：

> 当前总价值 = 当前立刻拿到的奖励 + 折扣后的最佳未来价值。

如果进入 terminal state，就没有未来项：

$$
Q(s,a)=R(s)
$$

### 10.1 Bellman 方程例子

求：

$$
Q(2,right)
$$

从 `state 2` 向右会到 `state 3`。`R(2)=0`，`\gamma=0.5`。在 `state 3` 中：

```text
Q(3,left)=25
Q(3,right)=6.25
```

所以：

$$
Q(2,right)=0+0.5\times\max(25,6.25)=12.5
$$

这和前面直接算 return 得到的结果一致。

## 11. 随机环境：Stochastic MDP

前面的 Mars Rover 默认动作完全可靠：让它向左就一定向左。

但很多真实环境不是这样。机器人轮子可能打滑，风可能把飞行器吹偏，动作结果具有随机性。

假设：

- 命令向左时，90% 概率真的向左。
- 10% 概率误向右。

这就是 **stochastic environment（随机环境）**。

在随机环境中，一条策略执行多次可能得到不同轨迹和不同奖励序列。因此我们不再最大化某一次 return，而是最大化 **expected return（期望回报）**：

$$
\mathbb{E}\left[R_1+\gamma R_2+\gamma^2R_3+\cdots\right]
$$

随机环境下 Bellman 方程也要加入期望：

$$
Q(s,a)=R(s)+\gamma \mathbb{E}_{s'}\left[\max_{a'}Q(s',a')\right]
$$

直觉不变：当前价值仍然是即时奖励加未来价值，只是未来状态 `s'` 变成了随机变量，所以要取平均。

## 12. 连续状态空间

Mars Rover 只有 6 个离散状态，但真实机器人通常有 **continuous state space（连续状态空间）**。

例如一辆自动驾驶小车的状态可能是：

$$
s=[x,y,\theta,\dot{x},\dot{y},\dot{\theta}]
$$

其中：

- `x,y` 是位置。
- `\theta` 是方向角。
- `\dot{x},\dot{y}` 是速度。
- `\dot{\theta}` 是角速度。

直升机的状态甚至可能包含 12 个数：

- `x,y,z`：位置。
- roll、pitch、yaw：姿态。
- 三个方向速度。
- 三个角速度。

在连续状态空间里，状态不是 `1` 到 `6` 这样的有限编号，而是很多实数构成的向量。状态数量近似无限，无法用表格枚举每个 `Q(s,a)`。

这就是为什么后面需要用神经网络近似 Q 函数。

## 13. Lunar Lander：月球着陆器任务

本章实践作业是 **Lunar Lander（月球着陆器）**。

目标是控制一个模拟着陆器，使它安全降落在两个黄色旗子之间的着陆平台上。

### 13.1 动作空间

作业中动作是离散的，有 4 个：

```python
Do nothing = 0
Fire right engine = 1
Fire main engine = 2
Fire left engine = 3
```

动作含义：

- `0`：什么都不做。
- `1`：点火右侧发动机。
- `2`：点火主发动机。
- `3`：点火左侧发动机。

课程视频中也用简写：

```text
nothing, left, main, right
```

作业代码以 Gym 环境的数字编码为准。

### 13.2 状态空间

Lunar Lander 状态是 8 维向量：

$$
s=[x,y,\dot{x},\dot{y},\theta,\dot{\theta},l,r]
$$

含义：

- `x,y`：位置。
- `\dot{x},\dot{y}`：水平和垂直速度。
- `\theta`：倾斜角。
- `\dot{\theta}`：角速度。
- `l,r`：左右两个支脚是否接触地面，取 0 或 1。

### 13.3 奖励函数

作业和视频中的奖励设计包括：

- 越靠近 landing pad，奖励越高。
- 越远离 landing pad，奖励越低。
- 速度越慢越好，速度过快会受惩罚。
- 倾斜角越大越不好。
- 每条腿接触地面给 `+10`。
- 每次开主发动机扣 `0.3`，鼓励节省燃料。
- 每次开侧发动机扣 `0.03`。
- 安全着陆给 `+100`。
- 坠毁给 `-100`。

课程视频还提到，如果成功到达着陆区域，总奖励可能在 `100` 到 `140` 左右，取决于飞行质量。

这个奖励函数体现了强化学习的核心工程思想：你不写出每个状态下该按哪个按钮，而是把“想要什么、不想要什么”编码进 reward function。

### 13.4 任务目标

学习策略：

$$
a=\pi(s)
$$

使折扣回报最大化：

$$
R_1+\gamma R_2+\gamma^2R_3+\cdots
$$

Lunar Lander 中常用较大的折扣因子。视频里提到 `\gamma=0.985`，作业中使用：

```python
GAMMA = 0.995
```

因为着陆任务需要考虑较长时间后的结果，不能只贪图眼前奖励。

## 14. Deep Q-Learning：用神经网络学习 Q 函数

在连续状态空间中，不可能手动维护一个 `Q(s,a)` 表。DQN 的思路是用 neural network（神经网络）近似 Q 函数。

最直接的设计是：

- 输入：状态 `s` 和动作 `a`。
- 输出：`Q(s,a)`。

在 Lunar Lander 中：

- 状态 `s` 是 8 个数。
- 动作 `a` 可以用 4 维 one-hot 向量表示。
- 合起来输入 12 个数。
- 输出一个标量 `Q(s,a)`。

如果能训练出这个网络，那么在某个状态 `s` 下，可以分别计算：

```text
Q(s, nothing)
Q(s, right engine)
Q(s, main engine)
Q(s, left engine)
```

然后选最大值对应的动作。

---

## 15. 更高效的 Q 网络结构

课程随后改进网络结构：不再输入 `(s,a)` 输出一个 Q 值，而是只输入状态 `s`，一次性输出所有动作的 Q 值。

输入：

$$
s \in \mathbb{R}^8
$$

输出：

$$
[Q(s,0),Q(s,1),Q(s,2),Q(s,3)]
$$

这样在当前状态下只需要做一次神经网络推理，就能得到 4 个动作的价值。

这个结构也让 Bellman 目标中的：

$$
\max_{a'}Q(s',a')
$$

计算更方便，因为对 `s'` 前向传播一次就能得到所有动作的 Q 值，然后取最大值。

## 16. Bellman 目标如何变成监督学习标签

DQN 的训练思路是把 Bellman 方程变成训练标签。

经验元组是：

$$
(s,a,R,s')
$$

根据 Bellman 方程，希望：

$$
Q(s,a) \approx R+\gamma \max_{a'}Q(s',a')
$$

因此把输入设为：

$$
x=(s,a)
$$

把目标值设为：

$$
y=R+\gamma\max_{a'}Q(s',a')
$$

然后用均方误差训练神经网络：

$$
L=(y-Q(s,a))^2
$$

如果 `s'` 是终止状态，那么没有未来项：

$$
y=R
$$

这就是 DQN 作业中 `compute_loss` 的核心。

## 17. Replay Buffer：经验回放

强化学习中的数据不是独立同分布的。智能体连续执行动作时，前后状态高度相关。如果直接用连续经验训练神经网络，容易不稳定。

**Experience Replay（经验回放）** 的做法是：

1. 把最近经历的元组存入 memory buffer。
2. 每次训练时，从 buffer 中随机抽取 mini-batch。
3. 用这些随机样本训练 Q 网络。

这样可以：

- 减少连续样本之间的相关性。
- 让同一条经验被多次使用，提高数据效率。
- 使训练更稳定。

作业中使用：

```python
MEMORY_SIZE = 100_000
MINIBATCH_SIZE = 64
```

经验被保存为：

```python
experience = namedtuple(
    "Experience",
    field_names=["state", "action", "reward", "next_state", "done"]
)
```

## 18. Target Network：目标网络

直接用同一个 Q 网络计算当前预测和 Bellman 目标会产生问题：

$$
y=R+\gamma\max_{a'}Q(s',a';w)
$$

这里目标 `y` 也依赖当前网络参数 `w`。网络每更新一次，目标也跟着变，容易导致震荡和不稳定。

解决方法是使用 **target network（目标网络）**：

- `q_network`：当前正在训练的 Q 网络。
- `target_q_network`：用来计算目标值的较慢更新网络。

目标变成：

$$
y=R+\gamma\max_{a'}\hat{Q}(s',a';w^-)
$$

其中 `w^-` 是 target network 的参数。

作业使用 soft update：

$$
w^- \leftarrow \tau w + (1-\tau)w^-
$$

其中：

```python
TAU = 1e-3
```

这表示每次只把当前网络的一小部分参数混入目标网络，使目标变化更慢、更稳定。

---

## 19. Epsilon-greedy：探索和利用

训练早期，Q 网络还不可靠。如果总是选择当前 Q 值最大的动作，智能体可能永远不会尝试某些动作。

例如随机初始化时，网络可能错误地认为 main engine 永远不好。如果它永远不尝试主发动机，就永远不会学到主发动机有时是必要的。

**Epsilon-greedy policy（epsilon-贪心策略）** 的做法是：

- 以 `1-\epsilon` 的概率选择当前 Q 值最大的动作。
- 以 `\epsilon` 的概率随机选择动作。

随机动作叫 **exploration（探索）**。

选择当前最优动作叫：

- **greedy action（贪心动作）**
- **exploitation（利用）**

课程也提醒，`exploitation` 在强化学习文献里只是“利用当前知识”的技术术语，不是日常语义里的剥削。

作业中：

```python
E_DECAY = 0.995
E_MIN = 0.01
epsilon = 1.0
```

训练一开始 `epsilon=1.0`，几乎完全随机探索。随着训练进行，epsilon 逐渐下降到 `0.01`，后期主要利用学到的 Q 函数，但仍保留少量探索。

## 20. Mini-batch：更快的训练更新

如果 replay buffer 有很多经验，例如 100000 条，每次都用所有经验训练会非常慢。

Mini-batch 的思想是：每次只随机抽一小批样本，例如 64 条。

作业中：

```python
MINIBATCH_SIZE = 64
```

Mini-batch 更新的特点是：

- 每一步计算更快。
- 梯度方向有噪声。
- 长期平均仍然朝较好的方向更新。

这和监督学习中的 mini-batch gradient descent 是同一个思想。

## 22. 作业 Exercise 1：创建 Q 网络和目标网络

要求创建两个结构相同的网络：

- `q_network`
- `target_q_network`

1. 输入层：`Input(shape=state_size)`

- optimizer 是否是 Adam，学习率是否等于 `ALPHA`。

## 23. 作业 Exercise 2：`compute_loss`

`compute_loss` 输入一个 mini-batch：

```python
states, actions, rewards, next_states, done_vals = experiences
```

先用 target network 计算下一状态的最大 Q 值：

```python
max_qsa = tf.reduce_max(target_q_network(next_states), axis=-1)
```

Bellman 目标是：

$$
y_j =
\begin{cases}
R_j, & \text{if episode terminates}\\
R_j+\gamma\max_{a'}\hat{Q}(s_{j+1},a'), & \text{otherwise}
\end{cases}
$$

代码中利用 `done_vals` 一行实现：

```python
y_targets = rewards + (1 - done_vals) * gamma * max_qsa
```

如果 `done_vals=1`，说明回合结束：

```text
(1 - done_vals) = 0
```

未来项被去掉，`y_targets = rewards`。

如果 `done_vals=0`，说明还没结束：

```text
(1 - done_vals) = 1
```

目标就是：

```text
reward + gamma * max_qsa
```

接着取当前 Q 网络对已执行动作的预测值：

```python
q_values = q_network(states)
q_values = tf.gather_nd(
    q_values,
    tf.stack([tf.range(q_values.shape[0]), tf.cast(actions, tf.int32)], axis=1)
)
```

这里 `q_network(states)` 会输出所有动作的 Q 值，形状类似：

```text
(batch_size, 4)
```

但每条经验只实际执行了一个动作，所以要用 `actions` 把对应动作的 Q 值取出来。

最后计算均方误差：

```python
loss = tf.reduce_mean(tf.square(y_targets - q_values))
```

或者按照提示使用：

```python
loss = MSE(y_targets, q_values)
```

## 24. 网络更新：`agent_learn`

作业中用自定义训练循环更新 Q 网络：

```python
with tf.GradientTape() as tape:
    loss = compute_loss(experiences, gamma, q_network, target_q_network)

gradients = tape.gradient(loss, q_network.trainable_variables)
optimizer.apply_gradients(zip(gradients, q_network.trainable_variables))

utils.update_target_network(q_network, target_q_network)
```

这一段做了三件事：

1. 根据 mini-batch 计算 Bellman loss。
2. 对 `q_network` 参数求梯度并更新。
3. 用 soft update 更新 `target_q_network`。

`@tf.function` 装饰器用于提升训练速度。

## 25. 完整 DQN 训练流程

作业训练流程可以概括为：

1. 创建 replay buffer：

```python
memory_buffer = deque(maxlen=MEMORY_SIZE)
```

2. 复制初始权重：

```python
target_q_network.set_weights(q_network.get_weights())
```

3. 对每个 episode：

- reset 环境，获得初始 state。
- 在最多 1000 个 time steps 内循环。
- 用 epsilon-greedy 根据当前 Q 值选择动作。
- 调用 `env.step(action)` 获得 `next_state, reward, done`。
- 把经验存入 replay buffer。
- 每隔 `NUM_STEPS_FOR_UPDATE=4` 步检查是否更新。
- 若满足条件，随机抽 mini-batch 并调用 `agent_learn`。
- 如果 `done=True`，结束当前 episode。

4. 每个 episode 后更新 epsilon：

```python
epsilon = utils.get_new_eps(epsilon)
```

5. 如果最近 100 个 episode 平均分数达到 200，认为环境已解决：

```python
if av_latest_points >= 200.0:
    q_network.save("lunar_lander_model.h5")
    break
```

作业说明默认参数下训练大约需要 10 到 15 分钟。

## 26. DQN 为什么能逐步学好

一开始 Q 网络是随机初始化的，`Q(s,a)` 几乎是乱猜。

但算法会反复做：

1. 用当前策略和探索收集经验。
2. 根据 Bellman 方程构造目标值。
3. 训练网络更接近这些目标。
4. 用更新后的网络收集更好的经验。
5. 重复。

这有点像“自举”（bootstrapping）：模型用自己当前的估计构造新目标，再逐步改善自己。

这也是强化学习比监督学习更不稳定的原因之一：目标不是固定标签，而是随模型更新而变化。因此 target network、experience replay、soft update 这些技巧非常重要。

## 27. 强化学习的现实状态

课程专门提醒：强化学习很重要，但现实应用没有媒体宣传中那么普遍。

原因包括：

1. **仿真中容易，现实中难。**  
   很多论文在模拟环境或电子游戏中效果很好，但转到真实机器人会困难很多。

2. **真实世界数据昂贵。**  
   机器人试错需要时间，可能损坏硬件，也不容易像游戏那样无限重开。

3. **奖励函数难设计。**  
   游戏有分数，但真实世界不会自动告诉机器人“你做得好不好”。

4. **超参数更敏感。**  
   强化学习算法常比监督学习更难调。epsilon、学习率、buffer、target update 等参数稍微不合适，训练时间可能成倍增加甚至失败。

5. **应用数量少于监督学习和无监督学习。**  
   现实工程里，监督学习和无监督学习仍然是更常用的工具。

但强化学习仍然是机器学习的重要支柱，特别适合：

- 机器人控制。
- 游戏 AI。
- 需要长期决策的问题。
- 工厂调度和优化。
- 金融交易执行策略。
- 自动化决策系统。

## 28. 机器人访谈中的实践观点

最后一个视频是 Andrew Ng 和 Chelsea Finn 关于 AI 与 Robotics 的访谈。它不是公式推导，但很重要，因为它说明强化学习和机器人落地的真实难点。

### 28.1 机器人 demo 不等于通用能力

很多机器人视频看起来很惊艳，例如跳跃、跑酷、后空翻。但这些系统通常是在特定环境中精心调好的。

评价一个机器人 demo 时，应问：

- 如果移动一个物体，它还能工作吗？
- 如果初始位置变一点，它还能工作吗？
- 如果环境换成一个新房间，它还能工作吗？

真正困难的是 generalization（泛化）：让机器人适应不同物体、不同房间、不同场景，而不是只在固定 demo 环境里成功。

### 28.2 机器人缺少像 ImageNet 那样的大数据集

计算机视觉可以利用互联网上的大量图片和文本，但机器人控制数据很少。互联网上没有大量“如何移动机器人手指系鞋带”的现成数据。

机器人原则上可以自己收集数据，但这又带来新问题：

- 如何让机器人安全地试错？
- 如何让数据足够多样？
- 如何让实验自动 reset？
- 如何跨机器人平台共享数据？

### 28.3 Simulation-to-real 很难

仿真可以生成大量数据，但仿真物理和真实世界不完全一致。

例如：

- 摩擦力很难精确建模。
- 物体接触过程很复杂。
- 仿真时间步如果太细，计算成本很高。
- 真实世界物体和环境极其多样。

所以仿真是重要工具，但不能完全替代真实数据。

### 28.4 Meta-learning 和在线适应

访谈中提到 **meta-learning（元学习）**，也叫 learning to learn。它的目标是让模型不只是学一个固定任务，而是学会如何快速适应新任务。

例如机器人进入一个新厨房，要打开一个之前没见过的柜门。它可能需要试几次，快速适应这个柜门的形状和阻力。

元学习试图利用过去很多任务的经验，让模型在新任务上用很少数据快速调整。

### 28.5 强化学习不是只用于机器人

访谈也提醒：强化学习本质是 delayed rewards（延迟奖励）下的决策学习，不只适用于机器人。

它也可以用于：

- 教育系统中给学生代码作业反馈。
- 游戏测试和错误发现。
- 需要连续决策和长期目标的各种系统。

## 31. 本章易错点

1. **强化学习不是直接预测动作标签。**  
   监督学习学 `x -> y`，强化学习通常通过 reward 学策略，不需要每个状态的正确动作标签。

2. **Reward 和 Return 不一样。**  
   reward 是某一步的即时反馈，return 是一串奖励经过折扣后的累计结果。

3. **折扣因子越小，智能体越短视。**  
   `gamma=0.5` 会强烈偏好近处奖励；`gamma=0.99` 更愿意等待长期回报。

4. **Q 函数不是状态价值，而是状态-动作价值。**  
   `Q(s,a)` 评价的是在状态 `s` 先采取动作 `a` 的长期价值。

5. **最优策略可以由 Q 函数推出。**  
   在状态 `s` 选择 `argmax_a Q(s,a)` 即可。

6. **终止状态没有未来项。**  
   如果 episode 结束，Bellman 目标是 `y=R`，不能再加未来 Q 值。

7. **DQN 的输出是所有动作的 Q 值。**  
   作业中的网络输入 8 维状态，输出 4 个动作的 Q 值，不是输出一个动作。

8. **Target network 是为了稳定目标。**  
   如果目标值随当前网络剧烈变化，训练容易震荡。

9. **Experience replay 是为了打散相关样本。**  
   连续状态高度相关，随机抽样能让训练更稳定。

10. **Epsilon-greedy 中 epsilon 是探索概率。**  
    `epsilon=0.05` 表示 5% 随机探索，95% 选择当前最优动作。

11. **强化学习比监督学习更难调。**  
    它对 reward、epsilon、学习率、buffer、target update 等参数更敏感。

12. **仿真成功不等于真实世界成功。**  
    真实机器人有摩擦、硬件误差、环境变化、数据采集和安全问题。
